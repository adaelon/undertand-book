# ADR-0003 读时架构:独立阅读器 + 本地查询服务 + 可插拔 LLM 后端

状态:已接受(2026-06-21,grill 共识)

## 背景
第三层(消费)的产品形态定为**一个阅读器**,集成 ②确定性导航 + ③LLM问答。关键约束:预构建期与读时的 LLM 接入方式**必须分开**——预构建靠 agent harness,读时不能绑 harness。

## 决策
1. **预构建期**:跑在 agent harness(Claude Code / Codex 等)里,由 harness 提供 LLM,跑多 agent 流水线产出 `.understand-book` 产物。一次性、离线。
2. **读时**:阅读器是**独立产品,与任何 agent harness 脱钩**。plugin 启动一个**本地临时查询服务**(127.0.0.1,仅阅读时存活,U-A 启 Vite 同款),阅读器 = 瘦客户端 → localhost 路由。**V2 §4 端点 = localhost HTTP 路由**。
3. **③ 的 LLM 后端用户自选**(Anthropic / OpenAI / 本地 Ollama 等),藏在本地服务的 **provider 抽象层**后;②确定性检索 provider 无关。
4. API key / provider 配置留在本地 Node 进程,不进浏览器。

## 命门
读时 ③ 是计费 / 本地算力调用,成本随用户所选后端走;系统不绑定任何单一 provider。检索逻辑(②编排)留在 Node,不在浏览器重写。

## 否决
- **浏览器直连单一 provider(A)**:检索逻辑入浏览器、key 暴露、锁死 provider。
- **回连常驻 Claude Code 会话(B)**:阅读器无法脱离活 agent 独立使用,脆。
- **读时也绑 Claude Code/Codex**:与"阅读器独立产品"冲突,把消费门槛绑死在装了某 harness 的用户上。

## 影响
- 强化 [[ADR-0002]]:BYO 后端可能无 embedding 能力 ⇒ 更不能依赖向量,图谱检索是 provider 无关正解。
- 细化 [[ADR-0001]]:localhost 临时 server 属 plugin 本地形态,不违"不托管";§4 端点 = localhost 路由,§4 契约大体可保留(不必全重译)。
