# ADR-0004 引用准确率红线分层:结构红线(我们扛)+ 语义质量(逐后端度量透明)

状态:已接受(2026-06-21,grill 共识)

## 背景
[[ADR-0003]] 让读时 LLM 后端用户自选(可能满血 Claude,也可能本地小模型)。但 V2 §7.1 把"引用准确率 ≥95%"定为硬红线。"用户自选弱后端"与"系统保证 ≥95%"直接打架,责任不清。

## 决策
把 §7.1 红线("`citations[]` 中 LID **真实支持**断言的比例")拆成两层:

1. **结构红线(后端无关,系统强制,写进契约)**:`citations[]` 的 LID **只能从检索结果回填,不得由 LLM 自由生成文本**;确定性闸丢弃悬空 LID。⇒ 任何后端(含弱模型)都**无法捏造一个不存在的 LID**。这半 100% 守。
2. **语义质量(后端相关,度量 + 透明,不兜底)**:"LID 真支持断言"的达标率,用**金标准集逐后端实测**;阅读器显示"当前后端引用质量 N%",弱则提示换后端。我们提供度量,**不替用户选的弱模型兜底**。

## 命门
- 实现关键:citations **回填机制**(从检索结果挑,非 LLM 生成)——结构红线的物理基础。
- §7.1 措辞改:从"系统保证 ≥95%"→"**结构红线恒成立 + 语义质量逐后端实测并对用户透明**"。

## 否决
- **钉最低后端门槛**:违背 [[ADR-0003]] 的"用户自选"开放性。
- **红线仅对推荐后端成立**:把硬红线变成带星号的承诺,诚实但削弱契约。

## 影响
兑现 V2 §3(原文/补充结构隔离:citations 真 LID、model_supplement 无 LID)+ §11(金标准做客观锚)。与 [[ADR-0003]] BYO-LLM 配套:开放后端选择,同时守住引用不可捏造。

## S8 真跑回填(2026-06-23,glm-5.1,真基座 `game-programming-patterns` 小基座 40 节点)
金标准集 12 条(`crates/runtime/goldset/game-programming-patterns.json`,从已锚定 claim 确定性推导,期望=最小核心 LID),内层 `book.query` 逐条跑、确定性闸度量(`crates/runtime/src/goldset.rs`):
- **结构红线 = 100.0%(12/12)**:返回 citation 全是真 LID、**0 悬空**。结构红线物理基础(回填机制+确定性闸丢悬空)在真后端兑现,**决策1 验证**。
- **语义质量信号**:mean_recall **1.00**(每条都引到期望核心 LID)、mean_precision **0.75**、incomplete 0。precision<1 源于模型常额外引**真实邻近** LID(非悬空,多为同小节相关支撑),非捏造——印证「期望最小核心 vs 模型核心+相关」需人工评判额外引用是否真支撑(决策2「语义质量逐后端度量+人工裁」落地)。
- **金标准集规模**:12 条(4 章区各 2–4)对切片0 足够稳定区分结构红线;切片1+ 逐后端度量矩阵时扩。
- **语义质量阈值建议**(切片0 基线,推荐后端):recall ≥ 0.9;precision 仅作监控不设硬阈(额外真实引用不应判负)。最终阈值 + 人工试读认可由用户冻结。

## 何时回头
- 语义质量目标阈值(上方建议)+ 金标准集规模随切片1+ 逐后端度量矩阵细化。
- precision 是否升级为「额外引用人工标真支撑率」而非裸 precision(避免惩罚正确的相关引用)。
