# ADR-0007 阅读器命令优先、agent 可驱动(agent-CLI 普适);人机同命令面

状态:已接受(2026-06-21,grill 共识)

## 背景
[[ADR-0003]] 定阅读器为读时产品,[[ADR-0005]] 定 E 是读时书 agent。需求方追加硬要求:**阅读器里的 agent 必须能操作阅读器本身**,且阅读器要"做成 agent-CLI 普适"。

## 决策
1. **命令优先(headless core + thin UI)**:阅读器核心 = 一套**命令可寻址的引擎**;GUI 只是命令面之上的一层渲染,不是本体。
2. **全动作即命令**:人类能做的每个阅读器动作(翻到 LID、高亮、滚动、开概念/语境面板、记笔记、跳引用…)都暴露为**具名、可参数化、agent 可调用的命令**。
3. **人机同命令面、无特供通道**:E 及任意外部 agent(模块 D)通过**与人类 UI 相同的命令面**驱动阅读器。⇒ **在阅读器层重现 V2 原则三(单一接口、人机对称、无特供)**。
4. **agent-CLI 普适**:命令面对任意 agent 经 CLI / 程序化调用通用,不绑定我们的 E。

## 命门
阅读器**必须命令优先地建**,不能"UI 优先再补 agent 控制"——后者是返工。E 的 skills(见 grill 第 9 问)因此天然含 **reader-control 工具 = 这些命令**。

## 否决
- **UI 优先、agent 控制后补**:retrofit 等于重写。
- **agent 走特供控制通道**:违人机对称,且分裂出两套要维护的行为。

## 影响
与 [[ADR-0001]] 的"localhost 路由"统一:命令面是同一套,涵盖**内容查询(②③,只读基座)+ 阅读器控制(可变 UI/会话态)**。E 的工具集扩入 reader-control。

## 内部切分(2026-06-21 收口)
命令面 = 一套,**三命名空间**,对应三种可变性级别:
- `book.*` — **只读内容查询**(②③:navigate / context / query / synthesize),映射只读基座。
- `reader.*` — **可变 UI / 会话控制**(翻到 LID / 高亮 / 滚动 / 开面板 / 记笔记)。
- `memory.*` — **记忆层读写**(recall / save,[[ADR-0006]])。

契约硬边界:`reader.*` / `memory.*` **不得写入** `book.*` 的只读基座。E 与任意外部 agent 经同一三命名空间命令面驱动阅读器。
