# ADR-0008 LID 切分:纯确定性 + 句级独立层 + 忠实块映射 + 分区不变式自检

状态:已接受(2026-06-21,工程层 grill 议题1 共识)

## 背景
工程层 grill 议题1。V3 §3.1/§3.2 定了"LID = 深度可变有序路径、全自动导入",但切分算法本身留白。翻 U-A:其 knowledge 模式只到 `article`(整文件)粒度,**无文件内切分** —— 段/句级切分是本项目净新设计(U-A 仅可借"骨架确定性切、LLM 只管语义"的纪律)。体检 B13 点名中文句切是工作量黑洞,且句一旦成 LID 节点即承载引用红线。

## 决策
1. **切分纯确定性**:LID 全程是源文件字节的纯函数,LLM **零介入切分**(只在 §3.6 语义边出场)。无标记格式(裸 txt)只得退化粗粒度,由自检闸标低置信。
2. **句 = 段下独立 LID 层**:树真长一层(`3.2.5.2` = 章3节2段5句2);切片0 停在段,切片1+ 长出句。句是 citation 与语义边的一等公民。
3. **存储 = 有序 n 叉树(承载语义层级)+ 物化路径排序数组(查点/邻接/范围)**。冻结只读 ⇒ 二分即 O(log n),**不引入 B/B+ 树**(再平衡/分页机制是死重)。物化路径**逐段数值比较**,非字典序。
4. **段边界 = 忠实映射源块标记**(md 段/列表项/引用块/代码块;epub p/li/blockquote/br),**不检测体裁**。诗行/对话轮天然落到块边界;裸 txt 退化到空行。
5. **中文句切 = 句末标点集 `{。!?……}` + 引号/括号配对保护**(未闭合对内不切)+ 分号默认不算句末 + 省略号整体当一句末;对**每个叶子块文本统一跑**,1..N 句自然产出,无体裁分支。
6. **切分自检闸 = 分区不变式**:全覆盖(每正文字节恰属一叶子)+ 无重叠 + 父 span ⊇ 子并集 + 同级数值递增 + LID 唯一;非内容字节须显式归类,**不准静默丢**。切句覆盖率 = 入叶子内容字节 / 总内容字节,目标 100%;裸 txt 退化档允许 <100% 但强制标低置信。

## 命门
- 物化路径必须**逐段数值比较** —— 字典序会把 `3.2.10` 排到 `3.2.9` 前。
- 句切**配对保护**是引用粒度干净的物理基础(对话/引文不被劈碎),直接支撑 ADR-0004 结构红线。
- 分区不变式是纯确定性切分**可证伪**的护栏,呼应 B2「确定性工具做判定」。

## 否决
- **LLM 辅助切分**(推断 txt 章节):破可复现,把不确定性灌进冻结基座,自检闸不再纯确定性。
- **句靠段内偏移寻址(非独立节点)**:句非一等公民,图谱/检索连不到句。
- **B/B+ 树存储**:为可变 + 磁盘分页而生;本项目冻结 + 全内存(MB 级),机制纯属死重。
- **纯标点正则句切**:引号内对话被劈碎,引用粒度变脏。
- **轻量自检(只校唯一+有序)**:漏字节/重叠不被发现。

## 何时回头
- 若离开「冻结文件 + 全内存」假设(跨用户书库 / 多 GB 磁盘索引,见 [[ADR-0001]] 何时回头)→ 重评 B+ 树 / LSM。
- 若实测发现忠实块映射在某体裁切得过粗/过细 → 在**确定性框架内**加块类型映射规则,不引入 LLM / 体裁检测。

## 影响
落实 V3 §3.1/§3.2/§3.3 的切分与自检闸口径。窗口/批次切分(议题2)直接消费排序数组的**范围切片**能力。新增术语:段 / 句 / 物化路径 / 分区不变式(见 CONTEXT.md)。
