# ADR-0009 窗口/批次切分:LID 子树窗口 + 双约束预算 + subagent 并发 + 边界选择性重叠

状态:已接受(2026-06-21,工程层 grill 议题2 共识)

## 背景
工程层 grill 议题2。书是一棵连续 LID 树、**无 import/wikilink**,U-A 的"按文件分批 + import 图 neighborMap"不可直接移植(详见 memory 参考 + [[ADR-0008]])。需在"连续 LID 区间 + 输入/输出双约束 + 全局目录顶替 neighborMap"下重做窗口/批次。预构建期绑 harness([[ADR-0003]]),并发可借 U-A subagent 模型。

## 决策
1. **窗口边界 = LID 子树(章/节)为基本单元**:超输入预算则在子树内 token 驱动细分、过小则合并相邻同级子树,不跨粗粒度结构边界(不跨卷)。细分切点**必须吸附最近 LID 边界(段/句),绝不腰斩句**(守 [[ADR-0008]] 分区不变式)。⇒ 局部边=章节内、长程边=跨章节,语义干净;复用 ADR-0008 排序数组的范围切片。
2. **窗口预算 = 输入硬闸 + 输出软闸双约束**。输入硬闸:正文 token ≤ 上下文窗口 × 安全系数 −(指令 + 全局目录 + 输出预留),物理放不下必拆。输出软闸:单窗口预期节点/边软上限,超则再拆。两数字留切片0实测。
3. **执行/并发 = 复用 U-A subagent 模型**:一窗口 = 一次抽取 subagent 调用;相邻小窗口可融合进一次调用、产出按原窗口 LID 区间拆回;默认 5 并发,留实测。跨窗口连接靠**全局目录**(非 neighborMap)。
4. **边界处理 = 语义边界零重叠 + 人工切点带重叠**。子树边界零重叠(跨边界关系 = 长程边走 Pass2);仅超大子树内部细分的人工切点加小 overlap,确定性闸去重。

## 命门
- 内部 token 细分必须 **LID 边界对齐**,否则破 ADR-0008 分区不变式 + 污染局部边两端。
- **全局目录是 neighborMap 的替代物**(书无 import);Pass1 沉淀目录、Pass2 带目录连长程边(详见议题3)。
- "借 harness subagent 并发"**仅限预构建期**;读时模块 E 自建运行时是另一回事([[ADR-0005]]),勿混。

## 否决
- **固定 token 窗口**:腰斩句/段/章,局部边落半句,锚定脏。
- **只输入硬闸(无输出软闸)**:模型在填满窗口里过量抽低质边,无质量护栏。
- **自建并发运行时(预构建期)**:重造 U-A 已验证轮子,且偏离 ADR-0003「预构建绑 harness」。
- **全窗口统一滑动重叠**:章节语义断点处也重叠,冗余抽取浪费 token。
- **零重叠全靠 Pass2**:紧邻人工切点的细关系被粗粒度目录漏掉。

## 何时回头
- 切片0 实测后填入:输入安全系数、输出软上限、并发数、overlap 量。
- 若实测发现章节大小极端不均导致窗口利用率低 → 调融合/细分阈值,不改子树对齐原则。

**S2 实测回填(2026-06-22,《游戏编程模式》3047 叶/≈174k tok,token=确定性近似估算 CJK 1·其余 0.25):**
- token 口径 = 确定性近似估算(零依赖、保守,误差由安全系数吸收);后端 Claude Opus 4.8(ctx 200k)。
- 默认 `12000/80` → 64 窗口、利用率 22.7%;全程 partition.ok、单叶超限=0、每窗 ≤ 硬闸。
- **三层约束**(完整预算扫描数据+根因见 `docs/实测-S2窗口利用率.md`):①软闸(中文 ≈57 tok/叶,默认下先触发)→ ②硬闸过大稀释(装箱峰值≈59% @ 硬闸≈4000)→ ③结构地板 34 窗口(子树对齐+不跨卷的刻意代价,加大硬闸不可突破)。
- **利用率低是子树对齐的预期成本、非 bug**;窗口预算终值的主锚 = S3 Pass1 抽取质量(锚定率)而非 token 利用率(「更满≠更对」)。子树对齐/不跨卷原则在任何调参下不变。
- 并发数(默认 5)/ overlap(切片0 零长程边、无)留 S3 实测。

## 影响
落实 V3 §3.6 的「窗口/批次参数留切片0实测」—— 策略已定、数字待测。议题3(语义边两遍抽取)直接消费本窗口模型 + 全局目录接口。新增术语:窗口预算 / 融合批次 / 边界重叠(见 CONTEXT.md)。
