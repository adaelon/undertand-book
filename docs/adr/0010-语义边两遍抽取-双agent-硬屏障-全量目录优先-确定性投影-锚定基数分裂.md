# ADR-0010 语义边两遍抽取工程化:双 agent + 硬屏障 + 全量目录优先 + 确定性投影 + 锚定基数分裂

状态:已接受(2026-06-22,工程层 grill 议题3 共识)

## 背景
工程层 grill 议题3。V3 §3.6 定了"Pass1 抽局部边+节点→沉淀全局目录,Pass2 带目录抽长程边"的**策略**,但工程接缝留白:两遍是否同一 agent、串行屏障如何与并发协调、全局目录的体积/降级/生成方式、节点锚定基数、两遍 prompt 契约。翻 U-A([[ADR-0009]] 背景):其 `file-analyzer` 是**单遍 + `neighborMap`** —— 单遍可行仅因代码有 import 图可确定性解析出跨批邻居。书无 import,neighborMap 不可用,全局目录顶替,而目录要等 Pass1 全做完才存在 ⇒ **两遍是被无 import 逼出来的,不是设计偏好**。

## 决策
1. **双独立 agent 定义**:`pass1-local-extractor`(输入=窗口正文,输出=`{nodes:实体/断言, edges:局部边}`)与 `pass2-longrange-linker`(输入=窗口正文+全量全局目录,输出=`{edges:长程边}`,不产节点)。职责/输入契约/输出 schema 均不同,各自 prompt 单一职责,**不用 mode 分支合一**。
2. **硬串行屏障**:`全部 Pass1(5 并发)→ 确定性 merge 沉淀完整全局目录 → 全部 Pass2(5 并发)`。屏障卡两阶段之间。**不做流水线增量目录**——长程边两端可跨到全书任意远窗口,Pass2 任一窗口必须看到全量目录,残缺目录会系统性漏边,且漏边永久烤进冻结基座无法补救。
3. **全局目录全量优先 + 超限降级阶梯**:目录 token ≤ 输入硬闸减项([[ADR-0009]])则把**完整目录**喂每个 Pass2 窗口(零漏召回);超限走阶梯——**先压缩**(去一行摘要,`{lid,type,name,summary}`→`{lid,type,name}`,~30→~10 tok/条,多数中大书拉回全量可行区、仍零漏)→ **压缩后仍超限才确定性预筛**(窗内实体去目录词法/概念匹配喂 top-K)。
4. **全局目录确定性投影生成**:Pass1 **不输出** catalog;目录由脚本从 merge 后 nodes **纯函数投影**(`{lid: occurrences[0]||source_lid, type, name, summary}`)。目录≡节点(同源),零幽灵条目;压缩档=投影时少取 summary。
5. **节点锚定基数按类型分裂**:**实体/概念 = 单节点 + 多锚点 `occurrences:[LID...]`**(跨窗口靠 `id=entity:{normalized-name}` 归并为一个身份,直接兑现 book.concept 的"出现列表");**断言 = 单节点 + 单锚点 `source_lid`**(`id=claim:{lid}:{slug}`,跨窗口不归并)。
6. **边 scope 打标 + prompt 命门**:边 = `{source, target, type, direction, weight, description, scope}`,source/target=**节点 id**(非 LID),`scope∈{local,long_range}`(Pass1 产=local、Pass2 产=long_range)。窗口正文**每段/句前缀 LID 标注**喂入,LLM 引用的 LID **只能回填、不自由生成**([[ADR-0004]] 结构红线的物理基础);Pass2 两端必须都是已存在节点 id,不新建节点。

## 命门
- **两遍是无 import 的必然产物**:有 import 图(U-A)单遍即可;书无 import ⇒ 目录必须先全沉淀 ⇒ 硬屏障不可回避。
- **目录确定性投影 ⇒ 目录与图谱强一致**;压缩/预筛都只是投影/检索的下游档位,不引入第二套真相。
- **实体多锚 vs 断言单锚**直接决定 merge 去重口径与 book.concept 实现;混为一谈会让 book.concept 出现列表碎裂或断言被错误归并。
- 窗口正文带 LID 标注是 Pass1/Pass2 全部锚定红线的**物理前提**——没有标注,回填无源,结构红线落空。

## 否决
- **单 agent + mode 分支**:两遍职责/schema 差异实质,合一让 prompt 带条件分支、两种产出混一契约。
- **流水线增量目录**:早跑的 Pass2 见残缺目录,系统性漏指向未完成窗口的长程边,永久烤进基座。
- **预筛优先(常态只喂相关子集)**:把"漏长程边"从大书边界情况降为**常态漏**;且预筛漏=最难判的低共现长程边漏。
- **目录二次分片(零漏但 M× 调用)**:为"压缩后仍超限"的极端大书付 N窗×M片 常态成本,买的是最低置信边;读时图谱遍历(V3 §3.6 末句)本就兜底,不值。
- **Pass1 LLM 显式输出 catalog**:与 nodes 信息重复、可能不一致(幽灵条目),多一个闸校验点。
- **统一单锚点(每次出现独立节点)**:同一实体跨窗口身份碎裂,book.concept 出现列表要运行时聚合同名节点。

## 何时回头
- 切片0 实测填:目录压缩触发阈值、预筛 top-K 的 K、压缩后仍超限的实际书规模分布。
- 若实测发现"压缩档(无 summary)"使 Pass2 长程边质量明显下降 → 重评保留精简 summary 或分层目录,不回退到二次分片。
- 若未来引入实体消歧(同名不同实体 / 异名同实体)→ 扩 `id=entity:{name}` 的归并键,届时重评 occurrences 合并规则。

## 影响
落实 V3 §3.6 的"两遍抽取+全局目录"工程机制(策略→可执行接缝)。**改 CONTEXT「锚定」**:实体/概念锚定一或多 LID(occurrences),断言/引用锚定单 LID。**新增术语**:Pass1/Pass2(两遍抽取)、occurrences(出现锚点集)、边 scope。议题4(确定性闸+merge)直接消费本 ADR 的交接面:闸校验①节点锚点 LID 真实存在②边两端节点存在③去重④悬空丢弃,复用 U-A `merge-batch-graphs.py` 归并骨架。
