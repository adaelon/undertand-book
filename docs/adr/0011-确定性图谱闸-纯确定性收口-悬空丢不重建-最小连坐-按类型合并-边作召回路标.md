# ADR-0011 确定性图谱闸 + merge:纯确定性收口 / 悬空丢不重建 / 最小连坐 / 二元闸三维正交 / 按类型合并 / 边只作召回路标

状态:已接受(2026-06-22,工程层 grill 议题4 共识)

## 背景
工程层 grill 议题4。[[ADR-0010]] 定了语义边两遍抽取并把交接面交给"确定性闸 + merge"(①节点锚点 LID 真实存在 ②边两端节点存在 ③去重 ④悬空丢弃)。本 ADR 把闸/merge 的粒度、职责边界、是否保留 LLM 复核钻实。参照 U-A `merge-batch-graphs.py`(确定性机械修复)+ `assemble-reviewer.md`(LLM 语义兜底)。关键差异:我们的 LID 是确定性切分的纯函数产物([[ADR-0008]]),全集已知且冻结 ⇒ "锚点真不真实"确定性可判,不需 LLM 猜;且引用红线([[ADR-0004]])禁止 LLM 制造未验证锚点。

## 决策
1. **悬空纯确定性丢弃,绝不重建**:边端 LID 查 LID 全集——存在则合法,不存在=LLM 幻觉锚点,确定性丢并记报告。**不学 U-A 让 LLM 用 default 值重建"本应存在"的节点/边**(那是制造未验证锚点,直撞 [[ADR-0004]])。
2. **最小连坐**(悬空粒度分层,不误杀合法身份):
   - 边 source/target 指向不存在节点 → **丢边,不连坐节点**;
   - 断言 `source_lid` 悬空 → 丢断言节点 + 连坐其边;
   - 实体/概念 occurrences **部分** LID 悬空 → **只剔该 occurrence,节点凭剩余真锚存活**;
   - 实体/概念 occurrences **全部**悬空 → 丢节点 + 连坐其边。
3. **闸纯二元 + 三维质量正交**:闸只判锚点真实性(真→留 / 悬空→丢),**不自产"低置信"档**。三个质量维度正交、不混:**锚点真实性**(闸,二元)⊥ **切分 confidence**(切分层 LID 级属性,[[ADR-0008]],节点/边/citation 沿锚定继承、读时透传)⊥ **关系 weight**(Pass 的 LLM 给,0~1)。低置信 LID 仍是真实 LID,闸不因低置信丢弃;低置信节点照进全局目录、照被 Pass2 连边,只继承标记。
4. **节点去重按类型合并(改 U-A keep-last)**:实体/概念 id 撞 → **occurrences 取并集**(name/summary keep-last),保 book.concept 出现列表完整;断言 id 撞(罕)→ keep-last;边去重 `(source, target, type, direction)` keep higher weight(沿用 U-A)。**U-A 的节点 keep-last 对多锚模型是 bug(静默丢锚),必须改**。
5. **砍 LLM reviewer,纯确定性 merge+闸 收口**:`Pass 产出 → 确定性 merge+闸 → 最终图`,无 LLM 收尾。U-A reviewer 三职责我们都不需要或确定性可做(①悬空 LLM 重建—已否决 ②补跨批边—Pass2 显式抽长程边,书无 importMap ③未知类型重映射—节点类型闭集 `{entity,concept,claim}`,异常确定性处理)。构建期不设语义检查的代价(单条"锚点真实但关系判错"的边)由读时机制兜(见决策6 + 命门)。
6. **边在读时只作召回路标,edge.type 不当 LLM 强先验**:图谱遍历用**边的存在**决定"捞哪些 LID 原文进证据集";**关系类型让 LLM 从捞回的原文现判**,不把 edge.type 塞进推理 prompt 当已认证事实。type 退居召回提示 / UI 展示 / 图谱导航。(读时设计原则,根在议题4,牵议题6/读时章节。)

## 命门
- **错边 ≠ 错引用**:语义边在读时是召回路标,不进 citations 生成;结构红线([[ADR-0004]])把 citations 焊死在"真 LID + 真原文",错边碰不到引用真实性层,最坏只污染检索精度(多捞噪声/漏捞)。
- **两道读时兜底消化错边**:①锚定推理 over 真原文([[ADR-0003]] 原则二)——无关原文大概率被 LLM 过滤;②`scope=auto` 证据不足逐级外扩([[ADR-0002]])——漏捞被补。残留(弱后端被噪声带偏)落在 [[ADR-0004]] 语义质量逐后端度量射程内(此处金标准集合法相关,因残留发生在读时输出层)。
- 决策6 把"方向错边(builds_on↔contradicts)"这条最重破坏路径**压平为纯 scope 噪声**——type 不作强先验,判错也只多捞/漏捞,不形成错误关系框架。
- **分区不变式闸([[ADR-0008]])是图谱闸的前置**:它先确立"LID 全集"这个真相源,图谱闸才能拿它判悬空。两闸串行不重叠——切分闸管"LID 自身合法",图谱闸管"图谱锚定到合法 LID"。

## 否决
- **U-A 式 LLM 恢复兜底**:用 default 重建未验证锚点,撞 ADR-0004。
- **节点级粗连坐**(任一锚悬空丢整节点):实体一个 occurrence 写错就杀掉它所有正确边,浪费多锚模型。
- **闸自产"证据弱→低置信"档**:要么 LLM 自评(违 B2),要么与 weight 重复;闸该只吐二元。
- **节点 keep-last(沿用 U-A)**:跨窗口同一实体只留最后窗口的 occurrences,静默丢真锚,book.concept 残缺。
- **保留 LLM reviewer 做语义兜底**:多一道非确定性环节;构建期 LLM 审 LLM 的边正确性正是 B2 警告的"AI 输出 + AI 评分自循环";其职责我们已机制层消除。
- **读时拿 edge.type 当强先验**:把可能判错的中间标签当权威事实喂 LLM,使方向错边升级为错答案。

## 何时回头
- 切片0 实测填:图谱锚定率(V3 §7.2 ≥90%)实际值、悬空边/节点丢弃率;若丢弃率异常高 → 排查 Pass prompt 的 LID 标注/回填是否失效,而非放宽闸。
- 若金标准集回归测出某后端引用质量系统性低,且定位疑似构建期错边密集 → 届时再评"只读标记、不改图的可疑边报告"(B 方案),而非现在预设。
- 若读时实测发现"完全不用 type 做检索优先级"导致召回效率低 → 可让 type 参与**召回排序**(仍不进推理 prompt),不回退到强先验。

## 影响
- **两次 merge+闸**(同一套逻辑两次调用):merge-1(Pass1 后)= 节点去重合并 + 局部边去重 + 闸 → 干净中间图 → 确定性投影全局目录;merge-2(Pass2 后)= 长程边并入中间图 + 边去重 + 闸 → 最终图。
- **闸失败可观测性报告**(沿用 U-A stderr 结构):丢弃悬空边(按 missing-source/target/both)、丢弃节点(断言锚悬空/实体全悬空)、剔除的 occurrence、去重合并数、低置信节点/LID 计数、图谱锚定率。NEVER 静默丢——每条丢弃可见。
- **merge 脚本职责边界**:**复用 U-A**(ID 规范化思路、边引用重写、悬空丢弃、stderr 报告结构、edge dedup);**净新**(节点按类型合并 occurrences、LID 锚点真实性校验查全集、occurrence 级剔除、低置信透传);**不要**(tested_by linker、importMap recovery、func/class id 重建、complexity 规范化)。
- 更新 CONTEXT:新增"确定性图谱闸",补"语义边"读时角色(召回路标)。回填 V3 §3.6(图谱闸细节)、§4.1(边作召回路标)。议题5(语境胶囊)消费本 ADR 的最终图。
