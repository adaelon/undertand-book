# ADR-0012 语境胶囊降格:砍物化产物,改为 book.context 读时确定性投影

状态:已接受(2026-06-22,工程层 grill 议题5 共识)

## 背景
工程层 grill 议题5。V3 §3.5 把"语境胶囊"列为模块A预构建产物(每 LID 预组装"前置背景 LID + 同主题关联 LID + 概念锚点"),供 `book.context`(§4.1)毫秒级返回 near/mid/far。此条目自 V1/V2 顺延,工程层从未审过其"物化必要性"。grill 先定了"若保留,则纯确定性投影、LLM 零介入"(否则撞 [[ADR-0004]]/[[ADR-0008]] 的物理约束),随即逼出更上游的前提质疑:既然图谱遍历本就毫秒级([[ADR-0002]] 砍向量的立论基础),把语境预物化进冻结基座的增量价值是什么?

## 决策
1. **砍掉"语境胶囊"作为物化产物**:模块A 不再预组装、不再烤进 `.understand-book` 基座。
2. **`book.context` 改为读时确定性投影函数**:`book.context(lid, granularity)` 读时现场从「议题4 最终图([[ADR-0011]])+ LID 物化路径树([[ADR-0008]])」纯函数投影 near/mid/far,复用 `scope=auto`([[ADR-0002]])的同一套图谱遍历,不引入第二套真相。
3. **一切基础是图谱本身,不在其上叠加派生抽象层**(需求方裁决:避免再加抽象图引入错误几率)。

## 命门
- 图谱已是检索结构([[ADR-0002]]),遍历毫秒级、全内存(MB 级,[[ADR-0008]]);物化语境 = 在检索结构之上再造一层冗余视图。
- 物化会新增"胶囊↔图谱一致性"维护负担与幽灵条目风险源,直接违 [[ADR-0011]] "目录≡节点、零幽灵条目"取向。
- 预存"第一跳邻域"与 `scope=auto` 读时动态外扩机制重复——同一件事做两遍。
- 真正贵的是 ③LLM 调用(秒级);物化省不掉它,顶多省一次毫秒级遍历,不值物化成本。

## 否决
- **保留物化胶囊产物**(原 V3 §3.5):冗余抽象层 + 一致性风险 + 与 scope 外扩重复 + 占基座体积;增量价值被"图谱遍历已毫秒级"抵消。
- **LLM 生成胶囊**(议题5 第一叉已否决):自由生成 LID = 未验证锚点,撞 [[ADR-0004]];不可复现烤进冻结基座,撞 [[ADR-0008]]。
- **混合(确定性骨架 + LLM 摘要文字)**:LLM 文字部分仍需复现/一致性处理,且若进检索路径则风险回流;骨架已蕴含距离(scope 编码),叠加换不来新区分力。

## 何时回头
- 若切片0 实测发现某一档投影是**全图扫描**、现算超毫秒级(如"全书与本 LID 同主题的所有 LID"在稠密大图)→ 把**那一档**预算成确定性索引(局部优化),不复活整个胶囊产物。
- 参照:U-A knowledge 模式无预组装语境胶囊,读时直接遍历图谱(memory 参考)——降格后与 U-A 形态一致。

## 影响
- **改 V3**:§1 原则二固化清单删"语境胶囊";§2 交付矩阵模块A删"语境胶囊" + 加降格注;§3.5 产物清单"语境胶囊"条改为"语境 = `book.context` 读时投影,非产物";§4.1 `book.context` 注明读时确定性投影(非读预存胶囊)。
- **改 CONTEXT**:新增"语境胶囊"词条 = `book.context` 读时确定性投影口径,点破其不再是物化产物。
- **下游待钻(议题5 续)**:near/mid/far 三档各自的精确投影规则——本 ADR 只定"读时确定性投影"框架,三档定义留下一叉。
- 与 [[ADR-0002]] 同构:图谱即检索结构,不在其上物化派生视图。
