# ADR-0013 book.context 三档投影:near/mid/far 确定性遍历 + 累积分层 + top-K 截断

状态:已接受(2026-06-22,工程层 grill 议题5 共识)

## 背景
[[ADR-0012]] 把语境胶囊降格为 `book.context` 读时确定性投影,只定了"读时投影"框架,三档(near/mid/far)的精确遍历规则、累积/互斥语义、体积控制留作下游。本 ADR 钻实。硬约束:一切基础是图谱 + LID 树,不叠派生抽象层(需求方裁决,见 [[ADR-0012]])。

## 决策
1. **三档保留为具名图谱快捷方式**:`granularity∈{near,mid,far}` 是面向消费者的意图档位,对内各焊死一条确定性遍历规则(非新数据结构):
   - **near** = anchor 的 LID 树邻接(前驱/后继兄弟段·句 + 父 + 直接子)∪ `scope=local` 边直连节点的锚 LID。
   - **mid** = 经 anchor 所挂概念/实体节点**二跳**——这些节点的**其他 occurrences** LID(同主题别处)。
   - **far** = `scope=long_range` 边,一端锚在 anchor(或其概念节点)、另一端的 LID。
   ⇒ 三档全 reduce 到四种已固化图谱原语:树邻接 / local 边 / 概念 occurrences 二跳 / long_range 边。
2. **累积半径 + 分层标注**:`granularity=far` 返回 near∪mid∪far 全部,`=near` 只算到近层即停。每个返回 LID 标 `{lid, layer, via}`——`layer∈{near,mid,far}`,`via` = 接入路径(树邻接 / local 边 type / 概念名 / long_range 边 type)。⇒ 累积"一次到位" + 分层"可分视图/可挑层",与 `scope=auto` 半径递增外扩同构([[ADR-0002]])。
3. **每档确定性 top-K 截断**,排序键全确定性:
   - near 按 LID 物化路径距离(越近越前);
   - mid 按共享概念/实体数(越多越前),次级 LID 距离;
   - far 按边 `weight` 降序,次级 LID 距离;
   - K 留切片0实测。排序用 `weight` 是固化属性的**召回排序**,不进推理 prompt(承 [[ADR-0011]] 何时回头),不违 B2。
4. **命令职责分工**:`book.context` = 截断的语境概览(top-K,快速定位);`book.concept(name)` = 某概念**全量** occurrences(不截断,穷尽主题)。

## 命门
- 三档是图谱关系的具名分类,**不是第三套抽象**——via 标注复用 [[ADR-0011]] "边作召回路标":type/概念名退居召回提示与 UI 展示,不当推理强先验。
- 截断是导航辅助的取舍,**不碰引用红线**:`book.context` 不产 citations;漏掉的低优先关联由 `book.concept` 全量兜底。

## 否决
- **互斥分区(三档不相交,要全得调三次)**:far 不含近处上下文反直觉,调用方还要自己拼。
- **不截断返回全部**:稠密图 mid/far 上百 LID,`book.context` 退化成数据倾倒,违"语境概览"定位。
- **granularity 砍成裸图谱遍历参数(hops/edge_scope)**:把图谱拓扑认知负担外推给每个调用方,违命令面对人友好。
- **LLM 排序/筛选**:违 B2 + 不可复现;确定性排序键(距离/共享数/weight)足够。

## 何时回头
- 切片0 实测填:每档 K 值;mid 二跳经超热门概念(全书高频实体)是否需按概念稀有度加权,避免热门概念淹没。
- 若实测某档读时遍历(尤其 mid 二跳)在最大书上超毫秒级 → 按 [[ADR-0012]] 何时回头,把该档预算成确定性索引(局部优化),不复活整个胶囊。

## 影响

- 更新 CONTEXT「语境胶囊」词条补三档规则。
- **议题5 闭环**。下游议题6(命令面命令签名)消费本 ADR 的返回结构 `{lid, layer, via}` 定 `book.context` 的 schema。
