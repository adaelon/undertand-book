# ADR-0019 构建侧增量(议题8 第一叉/8a):Merkle 双轨变更检测 + 三级分级 + Pass2 受影响追踪·读时兜 + 独立基座内容寻址

状态:已接受(2026-06-22,工程层 grill 议题8 第一叉共识)

## 背景
议题8(增量/版本)。体检 §9 / [[ADR-0009]] / V3 §3.4 已焊死「**书内容大改 → 当新书重建、不覆盖旧基座**」(理由:书文本不可变、基座冻结只读可分发)。但它**留一处真空**:书一个字没改、是**我们的构建器升级了**(切分算法 / 抽取 prompt / 模型版本)——这时也全量重抽整书图谱吗?LID 是文本纯函数([[ADR-0008]]),文本没变 LID 必不变,全量重抽是浪费。本叉(8a)钻**构建侧增量**:复用单元、变更检测、Pass2 连坐、产物身份。需求方提出参照 **Merkle 树**(快速判定超大数据是否相同 + 定位变更块)。最高原则(正确性优先、只喂确定性事实+真原文)贯穿。议题8 第二叉(8b)= 记忆迁移,另叉(承本叉产物身份)。

## 决策
1. **复用单元 = LID 子树/窗口**:per-窗口复用(对齐 [[ADR-0009]]「一窗口=一次抽取调用」,复用粒度天然落在 LLM 调用边界)。否决整书级(复用率 0)与 per-节点级(语义抽取产物是「整窗口一次 LLM 调用」的联合输出,粒度不可能细于调用)。
2. **双轨变更检测**(两类变更源正交,分轨编码):
   - **轨①文本 = Merkle 树**:`content_hash = H(本节点直接文本 ⊕ 有序子节点 content_hash)`,沿 LID 树聚合([[ADR-0008]] LID 本身即有序 n 叉树 ⇒ 天然 Merkle)。`merkle_diff(old_root,new_root)`:根哈希相同则整书剪枝;不同则沿树**下钻只触碰变更路径**,O(log n) 定位变更子树。改一个错字 → 只有那个叶子块哈希变。
   - **轨②构建器 = 层版本戳**:`build_manifest = {split_algo_version, extract_prompt_version, model_id}`,**正交于 content_hash**。
   - **命门铁律**:`content_hash` **必须纯文本,绝不掺构建器版本**——一旦掺入,版本升级使全部哈希失效 → 退化全量,Merkle 白用。我们比 U-A `fingerprint.ts`(per-file flat)更适配:内容是层级树,可树形剪枝。**不引入 rsync rolling hash**:块边界=LID 确定性切分(文本纯函数),插入不致块漂移,只改该块自身哈希,无需滑动窗口。
3. **变更分级 NONE/COSMETIC/STRUCTURAL 确定性合成**(无 LLM 判定,守 B2):
   ```
   split_algo_version 变            → STRUCTURAL（重切 LID + 重抽图谱，LID 变全下游连坐）
   merkle 命中 + 树形状变(增删段/章) → STRUCTURAL（该子树重切 + 邻接/全局目录/Pass2 受影响）
   merkle 命中 + 树形状不变(叶内改字) → COSMETIC  （LID 复用，只重抽该窗口图谱）
   prompt/model 变                  → COSMETIC  （LID 全复用，只重抽图谱层）
   以上皆否                          → NONE      （整窗口复用旧产物）
   ```
   Merkle 下钻天然区分「叶子内容变(树形状不变)」vs「子节点数/序变(树形状变)」⇒ 确定性算出三级。
4. **Pass1 增量 = 命中变更的窗口重抽**:仅 STRUCTURAL/COSMETIC 命中窗口重跑 `pass1-local-extractor`,其余复用旧产物。
5. **Pass2 长程边增量 = 三道确定性收窄 + 一道读时兜底**(破「目录变一条→全窗口 LLM 输入变→全连坐」):
   - **第一道·节点集指纹闸**:`window.catalog_hash = H(该窗口投影进全局目录的节点条目集)`。`∀window catalog_hash 不变 → 全局目录不变 → Pass2 全跳过`。⇒「改字不动节点」的 COSMETIC 挡在 Pass2 之外。**「窗口文本变」≠「Pass2 连坐」,闸门是节点集变没变**。
   - **第二道·目录 diff 分类**:节点删除 → 旧长程边悬空 → 确定性闸丢([[ADR-0011]] 已有机制,零 LLM);节点变更/新增 → 进入反向追踪。
   - **第三道·反向追踪受影响窗口**:变更/新增节点 N → 只重抽「N 所在窗口 + 旧图中与 N 已有长程边的对端窗口」Pass2,**非全书**。
   - **第四道·新机会边读时兜**:「旧窗口节点 ←→ 新增节点」的新机会边(对端旧窗口不在追踪集)接受**构建期漏**,由读时 `scope=auto` 外扩 + 锚定真原文兜底(复用 [[ADR-0011]]「长程边=召回路标、漏边读时可补」)。⇒ 增量基座长程边完整性略低于全量,但读时可兜,守正确性底线。
6. **产物身份 = 独立新 book_id 基座 + 旧基座只读并存 + 以旧为缓存 + 内容寻址去重**:增量是**生产优化**(拿旧基座未变窗口产物当计算缓存,LLM 不重跑),**产出独立新基座(新 book_id=v2),旧基座(v1)原封不动**(继续只读、可分发)。⇒ V3 §3.4「当新书重建、不覆盖旧基座」**字面落地,基座不引入 version 维度**。「每次增量存整份基座」的磁盘顾虑由**内容寻址存储/硬链接**化解(逻辑 v1/v2 独立、物理未变产物去重)。8b 据此 = 「v1→v2 跨基座 citation 重锚」。
7. **自检闸增量校验**(工程必然,无裁决):分区不变式([[ADR-0008]])、图谱锚定率([[ADR-0011]])是全局断言,增量后**只重验「变更子树 + 其与邻接的接缝」**,未变部分继承旧验证结果;新基座整体仍须满足全覆盖+无重叠+锚定率达标方可固化。

## 命门
- **双轨正交是全叉地基**:文本变(Merkle 可树形定位)⊥ 构建器变(全局层戳)。混轨即退化全量。
- **`catalog_hash` 闸是 Pass2 不连坐的关键**:把「Pass2 是否动」从「窗口文本变没变」收窄到「节点集变没变」,绝大多数小改 Pass2 零重抽。
- **增量 Pass2 不死守零漏 ≠ 违 [[ADR-0010]]**:[[ADR-0010]] 否决预筛是针对**首次构建**(常态漏低共现边);增量是**已有全量基座上补变更**,漏的仅「新增节点的新机会边」,且 [[ADR-0011]] 读时兜底正为此存在。两者不冲突,首次构建仍全量零漏。
- **增量=生产优化、非原地更新**:旧基座只读不动是 §3.4 与 [[ADR-0006]] 记忆隔离/可分发的前提,原地 patch 会同时破二者。

## 否决
- **整书级 fingerprint**(变了全量重建):复用率 0,等于不做增量。
- **per-节点/边级复用**:LLM 一次调用产整窗口联合输出,无法只重抽单节点。
- **`content_hash` 掺构建器版本**:版本升级全哈希失效,退化全量。
- **rsync rolling hash**:为块边界漂移而生;我们块边界=确定性 LID 切分,不漂移,纯属多余复杂度。
- **变更分级靠 LLM 判**:违 B2;Merkle 下钻 + 层戳确定性可算三级。
- **Pass2 全量重抽**(任一节点进出目录即全书 Pass2):「加一段话」也触发全书,增量在节点变更时失效。
- **增量也零漏**(新增节点强制全旧窗口重跑 Pass2):成本回到全量;[[ADR-0011]] 读时兜底使其不必要。
- **原地 patch 旧基座**:违 V3 §3.4「只读不覆盖」+ 破基座可分发 + 撞 [[ADR-0006]] 记忆隔离。
- **同 book_id + 基座内 version 维度**:把体检 §9 已砍的「版本」概念偷渡回基座层,违 [[ADR-0008]]「LID 一次冻结」,毁只读分发干净性。
- **阈值混合(变更比例切换追踪/全量)**:引入魔数 + 两套路径,两边不讨好。

## 何时回头
- 切片0 实测填:Merkle 分块粒度(到段还是到句)、内容寻址去重的存储后端、增量 Pass2「受影响追踪」实测漏边率(新机会边占比)、读时 scope 外扩对漏边的实际召回补偿率。
- 若实测「新机会边」漏召回显著拉低某后端语义质量([[ADR-0004]] 逐后端度量)→ 评估对「高变更密度增量」局部回退全量 Pass2,不预设阈值。
- 若离开「冻结文件+全内存」假设(跨用户书库,见 [[ADR-0001]] 何时回头)→ 重评内容寻址存储是否上对象存储。

## 影响
- **回填 V3 §3.4**:补「重建可走增量优化(以旧基座为缓存),产物仍是独立新基座、旧基座不动」一段;明确增量 ≠ 原地更新。
- **新增 CONTEXT 术语**:构建侧增量 / 双轨变更检测(Merkle 文本 ⊥ 层版本戳)/ 变更分级(NONE/COSMETIC/STRUCTURAL)/ catalog_hash 节点集指纹闸 / Pass2 受影响追踪 / 独立基座内容寻址存储。
- **承** [[ADR-0008]](LID 树=天然 Merkle、纯确定性切分、分区不变式)/[[ADR-0009]](窗口=复用单元、预算)/[[ADR-0010]](两遍抽取、全局目录、首次构建零漏不变)/[[ADR-0011]](确定性闸、长程边读时兜底)/[[ADR-0006]](记忆隔离、跨书对齐→8b)/[[ADR-0004]](引用红线、语义质量逐后端度量);参照 U-A `fingerprint.ts`(NONE/COSMETIC/STRUCTURAL 分级,我们换 flat→树形 Merkle)。
- **议题8 续**:第二叉 8b(记忆迁移:v1→v2 跨基座 citation 重锚),另落 ADR-0020。
