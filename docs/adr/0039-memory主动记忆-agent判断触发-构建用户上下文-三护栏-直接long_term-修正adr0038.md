# ADR-0039 memory ② 用户主动记忆:agent judgment 触发 + 构建用户上下文 + 三护栏 + 直接 long_term(修正 ADR-0038 决策2 计数器与认知诚实否决)

状态:已接受(2026-06-28,P4-3 §0.5 grill;落地实现留后续会话)

## 背景

[[ADR-0038]] 把 ② 用户主动 LLM 记忆的触发定为「用户显式记下 X」+「跨轮反复提及(LLM 识别,**基于确定性『提了 N 次』**)」,并在命门/否决里写「记『用户多次提到 X』(事实)**非**『用户对 X 有兴趣』(推断)」「否决 reader_profile 推断认知水平」。P4-3 实现前 grill,需求方提两点修正:

1. **确定性计数器效果差**:机械阈值 N 次抓不准真正值得记的东西。应让 **agent 在 E loop 读时凭自身 judgment 觉得「值得记」就主动调 memory 工具写下来**(像 Claude Code,用户体验过比 Codex 计数器好)——「用户不必每次提示 agent 要在这个时机做这件事」。
2. **memory 本质 = 对用户上下文的构建**:让 agent 持续理解用户(背景/偏好/关注/卡点),免去用户重复交代。死限「只转述事实、不许记对用户的理解」会**杀掉 memory 的功能价值**。用户拒斥的是 Codex 的**方式**(后台 / 不透明 / 不可控 / 机械计数),不是「构建用户画像」本身。

## 决策

1. **触发改 agent judgment(砍计数器)**:② 主动记忆触发 = (用户显式「记住 / 记下 X」)∨(agent 在 E loop **读时 judgment** 觉得值得构建进对该读者的理解)。**确定性计数器 + 阈值作废**(修正 [[ADR-0038]] 决策2 后半 +「何时回头」反复提及阈值条)。「反复提及」**不再是独立机制**,融入 agent judgment(注意到读者反复追问自会记)。

2. **「记什么」放宽到「构建用户上下文」+ 三护栏**:② 允许记**对用户的理解**(偏好 / 背景 / 关注 / 卡点,含一定推断),不再死限「只转述事实」。守住的不是「事实 vs 推断」,而是**三护栏**:
   - **透明**:落可见文件,用户随时看见记了什么。
   - **用户终裁**:可改可删,记错一句话纠正(承 [[ADR-0030]] 可撤销)。
   - **认知诚实 + 锚真 LID**:记理解时标清**基于什么证据**(「在 §3.2 反复追问所有权,像是卡在这」),不伪装铁事实、不凭空;citation 锚真 LID 过确定性闸。
   **放宽 [[ADR-0038]] 两条否决**:② 主动记忆允许带对用户的理解 / 推断(守三护栏);**确定性 reader_profile 派生(P4-2)仍不塞推断**(无 LLM 的确定性产物归确定性,[[ADR-0038]] 决策3 不变),两者分开。

3. **直接 long_term + 透明可删兜底(非提议态)**:agent judgment 记的**直接落 `long_term`**;靠「透明可见 + 随时可删」兜底纠错(**事后纠正 > 事前逐条批准**)。否决「提议态(`session`,用户保留才升 `long_term`)」——逐条把关违「免重复提示」初衷。区别于 [[ADR-0030]] agent 标注提议态(那是视图层 note/highlight 落 session;本条是上下文记忆,直接持久)。

4. **每条带时间戳 = 成长时间线**:② 记忆复用 `Record.generated_at`(已有,确定性、调用方注入);**memory 是学习成长轨迹**(今天卡点 → 后来读懂)。成长轨迹的**呈现 / 对比**(连成时间线给用户看)留 P4-4 四层文件,本刀不做。

## 命门

触发放给 agent judgment,但**三护栏不可松**——尤其透明 + 可删 + 认知诚实标注。区别于 Codex 的正是「**前台 judgment + 三护栏**」,不是「禁止理解用户」。一旦记的东西不透明 / 用户删不掉 / 伪装成铁事实,就退回 Codex 的不可控猜测。

## 否决

- **确定性计数器识别反复提及**([[ADR-0038]] 原决策2 后半):机械阈值效果差,抓不准真正值得记的。
- **死限 ② 只转述事实**:杀 memory 构建用户上下文的功能价值(用户明确反对)。
- **提议态(session,用户保留才升 long_term)**:逐条把关违「免重复提示」。
- **后台自动抽推断**([[ADR-0038]] 已否,仍否):agent judgment 是**前台读时**,非后台全量扫描。

## 何时回头(落地形态 PENDING,实现期定)

- **② 的 memory type 名**:候选 `insight` / `context` / `understanding`(区别于 `note`=用户逐字便签)。
- **citation 闸严格度**:推荐「无效 LID 确定性丢弃(承 [[ADR-0011]] 最小连坐)、不阻断整条、零有效 citation 仍可存」(content 是用户上下文,不像 book 答案必须有证据);待实现期确认是否要更严。
- agent judgment 记得太多 / 太碎 → 加 prompt 约束或 recall 时按 usage / 时间裁剪(**不复活计数器做触发**)。

## 影响

- **修正 [[ADR-0038]]**:决策2 触发(计数器作废 → agent judgment);命门 / 否决放宽(② 允许带对用户理解,守三护栏)。**保留**:透明四层产物、citation 锚定、用户终裁可删、确定性 reader_profile 不塞推断、读时主动非后台。
- 修 **CONTEXT「记忆 consolidation」**(② 触发 + 放宽 + 直接 long_term)。
- 重写**切片方案 P4-3**:**一条线**(不拆 a/b,计数器砍);落地形态 PENDING。
- 承 [[ADR-0015]](记录模型 + citation)/[[ADR-0030]](可撤销 + 会话边界)/[[ADR-0038]](memory 重定位)。
