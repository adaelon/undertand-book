# ADR-0042 预构建跨会话续建:逐窗原子落盘 + 存在性 + content-hash 校验 + agent 冷启动契约(无状态位)

状态:已接受(2026-06-29,预构建打磨 §0.5 grill;同日二次 grill 重定性为**跨会话 agent 续建**,非脚本崩溃恢复)

## 背景

预构建从没在真书上端到端跑过,而 build 是 **Claude(harness agent)在环驱动**的:真书 = 数十窗口 × Pass1 subagent 抽取,**一个会话的 token / 上下文根本跑不完**——token 耗尽 / 上下文撑爆 / 会话中断是**常态路径,不是异常**。当前 build 无中间产物落盘:`pass1-batch.ts` 一次性吃手工拼的单一 `outputs.json`、硬编码 `bookId`、末尾一次写。会话一死,在途抽取全丢、从头来。承软件工程准则 **A4「防上下文断裂」**:长任务必须拆成"只依赖文件状态、下一会话从零接手"的步骤。grill 收敛"中间产物保存 + 跨会话续建"的设计,逐岔口拍板。

## 决策

1. **续建 = 跨 Claude 会话,token 耗尽是常态**:续跑的真实场景 = 上个会话 token / 上下文耗尽后,**新 Claude 会话零上下文、纯靠磁盘中间产物接着建**(不是脚本崩溃恢复)。这是数十窗真书的**正常工作流**,不是错误兜底。

2. **逐窗原子落盘(命根)**:每抽完一窗立即原子写 `.understand-book/<bookId>/.build/pass1/<id>.json`(`{content_hash, nodes, edges}`),**会话可停在任意窗口、已抽全部幸存**。区别于旧"手工拼单一 outputs.json"(会话死 = 半成品全丢)。这是跨会话续建成立的物理前提。

3. **只物化"贵且不可重算"的 LLM 输出**:唯一持久化中间产物 = 上述 `pass1/<id>.json`。LID 树 / 窗口 / 输入正文 / 对比哈希全部用时从原书确定性重算、不落盘(承 [[ADR-0012]] 不物化派生;`windows.json` / 输入正文 txt 砍)。

4. **续建判定 = 存在性 + content-hash 校验,位置 id 键,无状态位**:`content_hash = sha256(buildPass1Input(window).text)`(`[lid]` 前缀正文,一哈希同锚 lid 集 + 真原文)。新会话重算窗口、逐窗比对:`pass1/<id>.json` 在且 hash 一致 → `done`;缺失或不一致 → `pending`。**刻意不引状态机(pending/done/failed 标记)/ watermark / lock**(承 [[ADR-0038]][[ADR-0039]] 砍单机过度工程);会话中断 / 半成品 = 没文件 = pending,二值。**位置 id 键(非内容寻址)**:跨会话续**同一本书**,重算窗口 id 必对得上,够用。

5. **agent 冷启动契约(写进 `skills/build/SKILL.md`)**:光有文件不够,需一份让陌生 Claude 自我定向的续建 playbook——`status <book>` 拿 pending → 逐窗 `emit-input` + subagent 抽取 + 原子写 → 全 done 跑 `pass1-batch` 收口。与项目 SESSION_CHECKPOINT(C4/C5 会话热启动)同招,只是冷启的是"构建状态"而非"会话状态"。

6. **本刀零 LLM、subagent 在环不变**:Pass1 抽取仍 subagent(`pass1-local-extractor`,[[ADR-0003]] 预构建绑 harness LLM、TS 侧从不自调 LLM)。本刀 = 包在抽取外的确定性脚手架:`emit-input`(现算喂抽取,不落盘)/ `status`(冷启动续建视图)/ `pass1-batch` 改造(消费 `.build/pass1/*.json` + hash 校验)。

7. **bookId 文件名 slug 派生 + 三处一致 + 非 ASCII fail-fast**:`deriveBookId(bookPath, override?)` 纯函数(emit-input / status / pass1-batch 共用,三处算同一 bookId 才找得到 `.build/`)。默认 basename 去扩展 slug(ASCII-safe:小写、空格 / 下划线→连字符、剥非 url-safe);`--book-id` 覆盖;**非 ASCII 主导 / 为空 → 报错要显式 id**,不建 CJK 目录坏工具链(承 [[windows-cjk-path-tooling]] + PB0 fail-fast)。

8. **收口拒绝残基座**:`pass1-batch` 遇 pending 窗口默认**拒绝收口** + 报 pending ids(缺窗 = 缺节点 = 图不完整);`--allow-partial` 显式兜底。

## 命门

`content_hash` 必须算自与抽取输入**同一份**正文(`buildPass1Input(window).text`)——否则陈旧静默复用,踩最高原则。**逐窗写必须原子且独立**(一窗一文件、写完即落),否则会话中断点不可靠、跨会话续建失真。`.build/` build-only,`Book::load` 绝不读(承 `long_range_candidates.json` 先例)。

## 否决

- **单一 outputs.json / 末尾一次写**:会话死 = 在途全丢,跨会话续建不成立。
- **`windows.json` / 输入正文 txt 落盘**:派生可重算,违 [[ADR-0012]]。
- **状态机 / watermark / lock / git-diff**:单机过度工程,承 [[ADR-0038]][[ADR-0039]]。
- **content-hash 当文件名**:跨会话续同书用不上(位置 id 够),且累积 orphan、不可读。
- **bookId 内容哈希派生 / CJK 目录名**:不可读 / 撞 Windows 工具坑([[windows-cjk-path-tooling]])。

## 何时回头

- **内容寻址续建**(书改了也复用未变窗口的抽取)= [[ADR-0019]] 计算缓存子集;本刀只续同一本书,内容寻址留此。
- 跨版本增量(LID 重映射 / citation 重锚)= [[ADR-0019]][[ADR-0020]],本刀不碰。
- Pass2 / discourse / formula 抽取是否也纳入同款续建(本刀只续 Pass1,先验形态)。
- 真书数十窗实测:`status` 重算窗口开销、单会话能抽几窗(token 预算)、id 漂移下 orphan。

## 影响

- `pass1-batch.ts` 去硬编码 bookId + 改消费 `.build/pass1/*.json`;新增 `deriveBookId` / `status` / `emit-input`。
- **`skills/build/SKILL.md` 补「跨会话续建 loop」**(agent 冷启动契约)。
- 切片方案补 **PB5** A4 子刀;CONTEXT 新增「构建工作区」「跨会话续建」术语。
- **解锁真书端到端预构建**:逐窗原子落盘 + 冷启动契约 = 数十窗抽取可跨多个 Claude 会话完成,不再"一会话跑不完 = 做不了"。
