## 🔴 地基级:三处自相矛盾,任一不解则全篇悬空

### B1. 「禁止运行时重推理」 vs `/query` `/synthesize` 推理代理 —— 整份文档的中心裂缝
- 原则二(L6)写死"运行时禁止重推理";但 `/query` 要 `evidence_chain` 推理步骤(L96)、`/synthesize` 是"跨LID语义编织与综合"(L103)、5.3 推理代理"面对多跳问题完成推理"(L139)。
- 一本书的预构建不可能穷举所有用户问题的答案。要么"禁止重推理"是空话,要么 `/query` 退化成只能吐预存片段的检索器。
- **方向**:重新定义"重推理"边界——预构建做*素材准备*(切分/索引/图谱/胶囊),运行时允许*基于素材的受限推理*。不重新定义,后面全是文字游戏。

### B2. 「人机对称消费,返回结构完全一致」(原则三 L7 / L100)不可能成立
- 人类要可渲染的阅读界面,Agent 要结构化 JSON。"完全一致"=要么逼人读 JSON,要么逼 Agent 解析 HTML。
- **方向**:真正想要的是"同一数据源 + 内容协商(content negotiation)",不是"同一字节流"。把绝对化措辞降级。

### B3. 「无状态」 vs Session / 人格 —— 概念打架
- 模块B 自称"无状态运行时调度"却又负责"外置Session管理"(L14);模块E 称"无状态认知代理人"(L125)又有人格模板。
- session 和人格都是状态。**方向**:统一为"计算无状态,状态外置存储";承认人格=运行时配置,不是玄学。

## 🟠 结构级:模块划分与路线图有依赖倒挂

### B4. Phase 1 让 D 在 C 之前完成,但 D 的准确引用依赖 C 的引擎(L15-16, L154-161)
- 模块D(开放接口)对外承诺"边界可见性 + 精确引用",可"精确引用引擎"在模块C(L15),C 在 Phase 2。D 凭什么在 Phase 1 返回准确引用?依赖倒挂。

### B5. 瀑布式"顺序铁律"放大风险而非降低(L145-169)
- 模块E 依赖 A/B/C/D 全部验收;"不可逆""任何跳跃导致不可交付"。任一环延误→全线阻塞,且最有价值的产品(E)最晚见光,反馈最晚到。
- **方向**:让 A→(瘦D)→瘦E 形成早期端到端验证链,而非四阶段串行。

### B6. 三代理 ↔ 三层API 是 1:1 映射,Agent 层的增量价值未被论证(4.x vs 5.3)
- 坐标代理=`/manifest`、语境代理=`/context`、推理代理=`/synthesize`。既然一一对应,模块E 相对裸API 多提供了什么?文档没回答。若只是"换人格说话",E 是薄壳,不配做独立阶段。

## 🟡 契约缺口:工程化必备项整篇缺失

### B7. 书的导入流程 = 0
epub/pdf/txt 怎么变成 LID 体系?模块A 的输入格式完全没定义(L19-29)。真正的工作量黑洞。

### B8. 错误模型缺失
LID 不存在 / 越界 / 版本不匹配 → 返回什么?所有 API 只写了 happy path。

### B9. 版本迁移协议缺失
`/limits` 有"坐标体系版本"字段(L111),但书一改版,已引用旧LID的Agent怎么办?没有迁移/兼容策略,LID"全局唯一地基"就是流沙。

### B10. `scope: auto` 无判定规则(L82)
auto 怎么决定 local/cross_chapter/global?没规则=不可实现。

### B11. 成本/延迟预算 = 0
"低延迟消费器""毫秒级查询"(L14, L27)有形容词无数字。每个 `/query` 背后几次 LLM 调用、多少 token、多少钱、p99 延迟目标多少?全空。

## ⚪ 指标空转 & 过度设计

### B12. 三个核心红线指标不可证伪(L27, L48, L100, L171-176)
- "图谱覆盖率≥90%" —— 分母是什么(实体数?句子数?人工标注集?)没定义就没法测。
- "引用准确率≥95%" —— "准确"的判定标准、谁测、抽样多少、有无 ground truth,全无。质量红线却不可执行。
- "Agent在≤3次调用内建立全书空间心智模型"(L48)——"心智模型"不可观测,纯文学修辞当验收标准。

### B13. LID 格式 `卷.章.节.段.句` 假设过强(L25)
诗集、对话录、论文集、无章节随笔都不符合。"句子级切分"在中文里(无空格、嵌套引号、省略号)是模块A最大隐藏工作量,被"可下探至句子"一句带过。

### B14. 人格"构建期注入"(L127)与现实冲突
构建期注入→一本书锁死一种人格?还是三种全预构建?且人格化输出与原则三"人机对称、结构一致"直接打架。人格本质是运行时 prompt,别硬塞进构建期。

### B15. 生物隐喻通胀
前庭觉/视觉/语言思维/认知肉身/数字感官/认知肉身人格化层——不增加工程信息,反而掩盖契约细节,让评审误以为想清楚了。工程 spec 应全部去隐喻化。

---

## 一句话体检结论

> 这份文档是一份**优秀的愿景宣言、危险的工程规格**。它的地基(原则二)与它的卖点(运行时智能问答)直接互斥;三条质量红线全部不可测;真正的工作量黑洞(书导入、句子切分、版本迁移)被一笔带过或完全缺席。
>
> **能动工的前提:先解 B1/B2/B3 三个地基矛盾,再补 B7/B12 两个黑洞。** 其余(模块拆分、隐喻、瀑布路线)属于可改善但不致命。

## 评审落定决策(grill 共识,2026-06-21)

> 以下 9 条为压测访谈中与需求方逐条确认的结论,直接覆盖/修正上文对应 B 项。
> 格式:决策(≤30字)+ 否决备选 + 命门/何时回头(仅非显然处)。

### §1 运行时推理哲学(修正 B1 / 原则二)
**决策**:运行时允许 LLM 推理,但必须锚定检索到的素材。
**否决**:
- 纯检索零推理:做不出"作者是否自相矛盾"类多跳问答。
- 无约束推理:脱离原文,引用准确率红线必崩。
**命门**:原则二须从"禁止运行时重推理"重写为"运行时推理必须锚定检索素材"。

### §2 素材不足时的行为(修正 B1 下游)
**决策**:允许 LLM 用外部世界知识补充,但必须显式标注。
**否决**:
- 诚实降级(只答原文支持部分):需求方要更强能力。
- 交给调用方决策:把负担推给消费者。

### §3 原文支持 vs 模型补充的结构隔离(修正 B12 / 原则一)
**决策**:`citations[]` 只放真 LID;模型补充另起 `model_supplement`(无 LID)。
**否决**:
- 哨兵值混流(lid=EXTERNAL):迟早被当真 LID 跳转。
- 仅靠 source 字段不隔离:依赖调用方自觉过滤。
**命门**:引用准确率≥95% 只测 `citations[]`,分母才干净可证伪;原则一加一句"仅约束 citations"。

### §4 LID 结构(修正 B13)
**决策**:LID 是深度可变的有序路径,非固定五级。
**否决**:
- 固定 `卷.章.节.段.句`:诗集/对话录/论文塞不进或生畸形坐标。
- 逻辑+物理双层分离:抽象过重。
**命门**:硬保证全局唯一 + 同级有序 + 可双向跳原文;下游不得假设固定段数。

### §5 书导入自动化程度(修正 B7)
**决策**:全自动导入,零人工。
**否决**:
- 机器草稿+人工校对 / 人工主导:需求方要规模化、拒绝人工环节。
**命门**:见 §6——零人工必须配确定性自检闸,否则质量红线崩。

### §6 LID 地基质量护栏(B7 配套)
**决策**:固化前插确定性校验闸,不达标自动打回或标低置信。
**否决**:
- 直接信任全自动产物:95% 红线沦为纸面。
- 抽样人工审计:重新引入人工,与 §5 冲突。
**命门**:校验项含 LID 唯一性/同级有序断言、结构自洽(无空层跳级)、切句覆盖率、图谱锚定率。呼应"AI 出产物、确定性工具做判定"。

### §7 模块 E 定位(修正 B6)
**决策**:保留模块 E,但必须论证其相对裸 API 的增量,否则降级。
**否决**:
- 原样保留(三代理=三API的薄壳):不配独立阶段。
- 直接降级为参考客户端:暂不一刀切。
**何时回头**:若 E 写不清编排/跨API状态/主动策略等增量 → 从关键路径摘下,降为示范 consumer。

### §8 人格归属(修正 B14)
**决策**:人格是运行时配置参数,模块 A 只读基座保持人设无关。
**否决**:
- 构建期烤三套:基座体积×3 且掺入消费者人设。
- 混合预构建:边界模糊。
**命门**:L127"构建期注入"改为"运行时选择"。

### §9 LID 版本演进(消解 B9)
**决策**:不存在版本迁移;LID 导入一次即冻结,新版书当新书重建。
**否决**:
- 迁移表(old→new):新复杂度,重映射可能丢节点。
- 多版本共存:书文本不可变,无需预设反复重建。
**命门**:发布后基座只读,换算法当新书建、不覆盖旧基座(流程纪律,非协议)。

### §10 实施路线(修正 B5 / B4)
**决策**:先打一条最薄端到端竖切(一本书+最小LID+最小图谱+只通 /context+/query),验证核心论点后再逐层加深。
**否决**:
- 原样瀑布"顺序铁律":最贵阶段(模块A全量)做完才第一次端到端反馈。
- 只修工序倒挂(D 挪到 C 后):依赖对了,但晚反馈的根本风险没解。
**命门**:与本方法论"小切片、早反馈"同向;B4 依赖倒挂在竖切里自动消解(首刀含一小撮 C 的引用能力)。

### §11 引用准确率度量(修正 B12 上半)
**决策**:小而稳的人工金标准集做回归测试客观锚(验收依据);未覆盖流量用 LLM-judge 仅作监控信号、不作验收。
**否决**:
- 纯人工金标准集:建集贵、覆盖窄,跟不上真实流量。
- 纯 LLM-judge:AI 答+AI 评,自循环,95% 无意义。
**命门**:呼应"执行结果做客观锚、LLM-judge 只在无客观锚兜底";图谱覆盖率(≥90%)分母待定(拟:有图谱锚定的 LID 节点 / 总 LID 节点,对金标准实体集测召回)。

### §12 查询成本/延迟取向(修正 B11)
**决策**:`/query` 分层——简单问题走快路径(1次调用,1-2秒),多跳/`/synthesize` 走深路径(多次调用,数秒),用 `scope` 分流。
**否决**:
- 一刀切极速:多跳问题答得浅,毁"会思考的书"。
- 一刀切质量:简单问题也烧十几秒+token,成本失控。
**命门**:先厘清两类延迟——图谱/索引查询(确定性,毫秒级)vs LLM 路径(秒级);文档把二者混叫"低延迟"是认知错误。

### §13 scope=auto 路由(修正 B10)
**决策**:检索广度自适应——从 anchor_lid 局部起,证据不足逐级外扩(节→章→跨章→全书)直到充分或触顶。
**否决**:
- 前置分类器:拍错范围整条路由错,且静默。
- 默认local+模型自请求升级:每次升级多一轮 LLM 往返。
**命门**:猜错范围被"证据不足"自动纠回,不依赖脆弱前置分类。

### §14 错误模型(修正 B8)
**决策**:标准化结构化错误契约——统一 `error_code`+`message`+可选 `nearest_valid_lid`,所有端点共用。
**否决**:
- 宽松降级(返回最近邻内容):静默给错数据,毒化引用准确率。
- 仅 HTTP 状态码:Agent 不知错因、不知下一步。
**命门**:错误是契约的一部分(与"单一完整接口"一致);Agent 靠机器可解析错误自我纠错,支撑 §13 自适应检索。

### §15 图谱覆盖率口径(修正 B12 下半)
**决策**:分两层——自检闸用确定性"节点锚定率(有图谱锚定的LID/总LID)";语义验收用"金标准实体召回"(同 §11)。
**否决**:
- 只用节点锚定率:锚定率高 ≠ 把书里概念抓全。
- 只用金标准召回:每本书建金标准贵,进不了自动闸。

### §16 生物隐喻处理(修正 B15)
**决策**:工程契约层去隐喻(导航层/单点查询/问答层等素词),愿景叙事层可保留诗意。
**否决**:
- 全删:愿景叙事损失产品表达力。
- 全保留:隐喻进工程契约,持续掩盖接口细节。
