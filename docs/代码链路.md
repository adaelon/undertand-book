## 2026-06-22 · S2 窗口切分(LID 子树窗口 + 双约束预算 + 确定性 token 近似)

**触达**:
- `packages/core/src/window.ts:splitWindows` — 按 LID 章/节子树切窗口:atomic 子树整窗、超限子树内逐叶贪心细分(吸附 LID 边界不腰斩)、过小由贪心累加自然合并、顶层间封口(不跨卷);单叶超硬闸自成一窗标 `overBudget`(诚实不腰斩)`[ADR-0009]`
- `packages/core/src/window.ts:estimateTokens` — 确定性近似 token(CJK `/[一-鿿]/` 按 1、其余 0.25,ceil),零依赖
- `packages/core/src/window.ts` — `WindowBudget`(maxInputTokens 硬闸 + maxLeavesSoft 软闸)/ `DEFAULT_BUDGET`(12000/80 占位,留实测)/ `Window`(leafLids/tokens/spans/overBudget)
- `packages/core/test/window.test.ts` — vitest 8 例:estimateTokens + 划分(无遗漏无重叠)+ token 一致 + 细分 + 过小合并 + 软闸 + 不跨卷 + 单叶超限不腰斩
- `skills/build/window-cli.ts` — S2 CLI(tsx):适配器→segment→splitWindows→窗口数/token 分布 + 窗口层划分校验,违例非零退出

**入口**:`pnpm exec tsx skills/build/window-cli.ts <epub|md> [maxInputTokens] [maxLeavesSoft]`(SKILL.md 编排第 2 段;消费 S1 `LidNode[]` + source)。
**测试**:`pnpm -C packages/core test`(17 例:S0 2 + segment 5 + epub 2 + window 8,全绿)+ `typecheck`(tsc 0)。
