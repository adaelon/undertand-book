# 切片方案 · S12 连续滚动阅读模型

> **触发**:S11 打磨后仍存在结构性阅读问题:reader 视口是 `anchor ± radius` 的中心化窗口,TopBar 上/下按钮一次移动 3 叶,导致正文几乎整屏替换、视觉中心跳到新窗口中段。  
> **冻结决策**:见 `docs/adr/0043-reader连续滚动视口-后端区间窗口-前端虚拟流-note-overlay.md`。  
> **边界**:本轮只改阅读流和相关视口契约;不新增后端全文搜索、不改 agent toolcalling 策略、不重做 memory 模型、不引入页码。

## 1. 目标模型

```text
Reader model:
  viewport = { top_lid, bottom_lid, anchor_lid, visible_lids, width }
  top_lid = 当前连续流区间起点
  bottom_lid = 当前连续流区间终点
  anchor_lid = 后端管理的新窗口中段叶,供 agent/state 共享
  visible_lids = leaf_lids[top..top+width]
  width = 默认 20 叶

Commands:
  scroll(delta):
    move top by delta leaves
    clamp to [0, leaf_total - width]
    visible_lids = new interval
    anchor_lid = midpoint of new interval
    mark all visible_lids as read

  goto(lid):
    resolve lid to leaf or subtree first leaf
    set top_lid = target leaf
    set visible interval from target
    anchor_lid = midpoint of interval
```

## 2. A1 切片总声明

- **做**:把 reader 从中心化叶序窗口改为连续滚动阅读模型:后端提供可配宽度区间 viewport,前端用原生滚动驱动虚拟流,notes 从正文 seg 循环拆出为 overlay,TopBar 移除翻页按钮。
- **不做**:不新增后端全文搜索、不改 agent toolcalling 策略、不扩 `/agent/chat` 的 top_lid 契约、不引入页码、不重做 memory 存储模型。
- **完成判据**:后端 reader/server 测试通过,前端 typecheck/build 通过,浏览器 smoke 能连续滚动阅读且无整窗替换闪烁,目录和 note source goto 后目标叶成为顶部阅读起点。

## 3. S12 小切片顺序

```text
S12a reader viewport interval semantics [Rust]
  做:
    - 扩展 reader `Viewport` 为 top/bottom/width/visible_lids/anchor_lid
    - Reader 状态从 anchor_idx/radius 改为 top_idx/width + 派生 anchor
    - `scroll(delta)` 平移 top,clamp 到两端
    - `goto_lid` 让目标叶成为 top
    - `scroll` 将新 visible_lids 区间记入已读账本
  不做:
    - 前端虚拟滚动
    - agent chat 契约扩展 top_lid
  判据:
    - `cargo test -p reader` 覆盖初始窗口、scroll clamp、goto top、区间已读
    - `cargo test -p server` 仍通过

S12b API DTO and progress wiring [Rust + Vue/TS]
  做:
    - server JSON 透传新的 Viewport 字段
    - `packages/web/src/api.ts` 更新 Viewport 类型
    - 前端进度从 anchor_lid 改用 top_lid 在 leafOrder 中的位置
    - 现有 loadWindow 先兼容新 visible_lids 批量加载
  不做:
    - 引入虚拟列表缓冲
    - 改 note overlay
  判据:
    - `pnpm -C packages/web typecheck`
    - `pnpm -C packages/web build`
    - 浏览器 smoke: 初始加载、outline goto、note source goto 仍可用

S12c virtual reader buffer [Vue/TS]
  做:
    - ReaderPane 改为原生滚动容器驱动的连续流
    - App 持叶缓冲和边缘阈值,滚到边缘触发 `reader.scroll(+/-N)`
    - 新批次追加/回收时保持滚动位置连续
    - 避免整窗替换导致的闪烁
  不做:
    - note overlay 拆分
    - 改后端 viewport 语义
  判据:
    - 浏览器 smoke: 滚轮/触控连续向下读,新叶从底部进入、旧叶从顶部离开
    - 快速滚动不出现整屏闪烁或明显接缝
    - `pnpm -C packages/web typecheck` + build 通过

S12d note overlay split [Vue/TS]
  做:
    - highlight 继续保留在 `renderSeg` 段内 `<mark>`
    - ReaderPane 内 note 卡从 seg 循环中拆出为独立 overlay/list 层
    - overlay 按当前 visible LID 集过滤 notes,锚定对应 `data-lid`
    - RightRail Notes tab 行为保持不变
  不做:
    - memory 结构变更
    - highlight overlay 化
  判据:
    - note 卡在连续滚动回收段落时不闪烁
    - note source 跳转和临时 quote 高亮仍可用
    - `pnpm -C packages/web typecheck` + build 通过

S12e remove page buttons + keyboard fallback [Vue/TS]
  做:
    - 移除 TopBar 上/下翻页按钮
    - ReaderPane/reader container 支持方向键与 PageUp/PageDown 滚动
    - Debug foldout 保留 raw goto 作为验收入口
  不做:
    - 新增替代翻页按钮
    - 改 outline / search 行为
  判据:
    - TopBar 不再暴露翻页按钮
    - 键盘滚动可用,不破坏输入框/textarea 焦点
    - 浏览器 smoke + typecheck/build 通过
```

## 4. S12 总判据

```text
正文阅读从中心化窗口替换变成连续滚动流
  ∧ 后端 viewport 表达 top..bottom 区间,默认宽度 20 叶
  ∧ scroll 平移区间并按 visible_lids 记已读
  ∧ goto 目标叶落为 top,目录/note source 定位语义自然
  ∧ 进度% 使用 top_lid
  ∧ 前端无整窗替换闪烁
  ∧ note 卡不污染虚拟列表段高估算
  ∧ TopBar 移除翻页按钮且键盘兜底可用
```
