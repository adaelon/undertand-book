### S9 · ModelAdapter 确定性产物 JSON 规范性硬化(鲁棒抽取 + 抽不到诚实报错)`[Rust]`

**整体思考(为什么这刀长这样)**:
LLM 在系统里只有两个入口,对应「确定性产物」与「agent」两类,规范化策略不同:
1. **确定性产物路径** = 内层 `book.query` 的 `complete() → ParsedResponse`。**要 JSON 规范**。
2. **agent 路径** = 外层 `chat() → AssistantTurn`。用**原生 tool-calling**(`tool_calls`),**不纳入** JSON-in-content 规范——多轮编排本就该用原生 function-calling。
确定性产物路径的现状已做到三层:① **要求 JSON**(`response_format:json_object` + `OUTPUT_CONTRACT` + 温度0);② **只认 JSON**(`strip_fence` 剥围栏后整段解);③ **值不信 JSON**——`query()` 拿到 citations 后确定性交叉验停 `citations⊆证据集`,悬空当场滤掉(ADR-0004 结构红线)。
即 **JSON 只负责「形状规范」,值的「正确性」永远由确定性工具再校验一遍**(ADR-0016 决策4 + ADR-0004)。
缺口只在第②层:仅剥围栏、未从散文杂质里抽 JSON 子串 → 模型在 JSON 外包散文 / 返回空串时 `from_str` 硬失败(S8 真跑遇 1 条 glm 偶发空响应即此)。

**prompt 维度(明确不改,及为什么)**:
本刀**不改 `OUTPUT_CONTRACT`,不向 LLM 告知"我们会抽取"**。理由:
- 强后端(glm)走 `json_object` 模式,输出被**强制为纯 JSON**,模型无法在外面包散文 → 鲁棒抽取在强后端几乎不触发,纯属**防御偶发**(空响应/漏网围栏)。`OUTPUT_CONTRACT` 已写明「只输出一个 JSON 对象」,无需再教模型抽取这回事。
- 「放宽 prompt、允许模型自由推理后我们抽 JSON」是**另一个决策**(think-out-loud / few-shot 提质),**与 `json_object` 模式冲突**,属 ReActAdapter / 弱后端 / 质量实验领域 → **切片1+**。
- 真正"prompt 要变"的是切片1+ **ReActAdapter**:弱后端不认 `json_object`,届时 prompt = ReAct 模板(显式要求把最终答案包成 JSON 对象),消费侧的鲁棒抽取才是主力。本刀是那条路径的**强后端最小前置防御**(ADR-0016 决策5)。

- **做**:把 `NativeAdapter::complete` 的 `strip_fence` 升级为**鲁棒 JSON 抽取**:剥围栏 → 若整段非合法 JSON,扫描抽取第一个**平衡 `{}` 对象子串**(跳过字符串字面量内的括号)→ 解析。**只消费抽出的 JSON,忽略前后散文杂质**。
- **不做**:不做 ReActAdapter 完整解析回填(注模板 / 多字段宽松容错)= 留切片1+;**不改 `OUTPUT_CONTRACT` / prompt**(理由见上,prompt 维度);**不动** `ParsedResponse` / 命令面契约 / 错误信封;不碰外层 `chat`(原生 tool_calls 不变);不放宽值校验(`citations⊆证据集` 确定性再校验不变)。
- **判据**:① 单测三形态都抽对(纯 JSON / 带 ```围栏``` / 前后包散文);② **抽不到合法 JSON / 空内容 → 显式 `PROVIDER_ERROR` 不静默成功**(守禁宽松降级 `[ADR-0015]`,不把拒答/报错当空成功——这是「抽取」与「静默吞掉」的分界);③ 现有真跑回归仍结构红线 100%。
- **触达**:`[ADR-0016]`(决策5 NativeAdapter 解析,ReActAdapter 前置最小版)+ `[ADR-0004]`(JSON 管形状、值确定性校验)+ `[ADR-0015]`(抽不到诚实报错 = 禁宽松降级)。**不需新 ADR**(现行决策已覆盖)。
- **实测落点**:glm-5.1 偶发空/啰嗦响应发生率;内层 `query` 是否也需加逐条重试(goldset runner 已有重试,内层裸调暂无)。
