# 切片方案 · 切片1 第一刀:前端阅读器 + localhost 服务连通

> **定位**:切片0 收尾(契约冻结基线 v1,V3 §6.1)后第一刀。把「阅读器前端页面 + 与后端连上」展开成 A1 切片声明 + A4 子切片。
> **配套**:架构决策见 `docs/adr/0028`(前端切片架构)+ `docs/adr/0030`(E agent 阅读器形态);被消费的冻结契约见 `需求文档-V3.md §4`;术语见 `CONTEXT.md`(命令面 REST 投影 / 连续正文渲染·LID 隐形 / 读位感 / agent 可撤销提议 / 读时会话边界)。
> **状态**:**已规划落档,未开工**(用户指定不在本对话做)。下次开工从 S10a 起。

## 0. §0.5 锁定决策摘要(全文见 ADR-0028 + ADR-0030)

1. **前端栈** = Vue/Vite SPA + localhost 服务(解 ADR-0021 PENDING);Vue app 落 `packages/web`。
2. **服务** = 新 `crates/server`,**tiny_http 同步**、线程每连接,`Mutex` 包单 `Book`+`MemoryStore`+`Reader` 会话(切片0 单用户单书);DAG 顶端依赖 read-tools/memory/reader/runtime/base-schema。同步:直调同步 `runtime::query`,不上 tokio(承 ADR-0024)。
3. **HTTP** = 命令面 1:1 REST 投影(`book.*`→GET、`reader.*`/`memory.*`→POST,端点名=命令名,错误透传 §4.4 信封)。
4. **不引 EPUB 阅读器框架**;薄 Vue 层把视口 `visible_lids` 渲染为**连续正文**,LID 是隐形接缝、被用时显形。
5. **无一等页码**;位置参照 = 视口当前 / 章节名(容器 LID)/ 概念内容;orchestrator 读模式每轮喂 viewport;UI 显**章节定位 + 进度%**。
6. **类型契约** = ts-rs 从 Rust 权威导出 API DTO 到 `packages/web`(承 ADR-0021,冻结契约的 TS 面自动生成防漂)。
7. **dev/打包** = Vite dev proxy(HMR)/ 打包 tiny_http 同端口服 `dist/` + API 单进程同源无 CORS(承 ADR-0022)。
8. **agent 形态**(ADR-0030,后补)= 前端对话主入口接**外层 E agent**(`/agent/chat`→`orchestrator::run`,reader 双向共享);agent 动作=**可撤销提议**(前端交互层,`OuterOutcome` 加 `effects[]`/`trace[]`,提议单元=一次对话回合);agent 标注落 `session` 层、用户确认升 `long_term`;五动作(goto 可返回 / highlight·note 询问保留 / 回答分屏+凝练成笔记或丢弃 / 查询踪迹可见);会话边界 = **用户手动「新对话」**(简化,不自动 idle 判定),新会话精炼上下文(`memory.recall` 兜底,完整轨迹摘要留 consolidation 刀)。

## 1. A1 切片总声明

- **做**:新建 localhost 服务(`crates/server`,tiny_http 同步)把冻结命令面投影成 REST + Vue/Vite SPA(`packages/web`)渲染 LID 窗口连续正文 + 闭环四动作(gotoLid/scroll/highlight/note)+ memory 标注回显,经 localhost 连通;读位感(章节+进度%);**前端对话主入口接外层 E agent(`/agent/chat`),agent 动作=可撤销提议 + 查询踪迹可见 + 回答分屏(详 ADR-0030)**。
- **不做**(明确排除):
  - ❌ 把 E agent loop 改成走 HTTP(agent 仍库内调用,HTTP 是同一命令面给「人」的另一投影)
  - ❌ 引 EPUB 阅读器框架(epub.js/foliate/readium)
  - ❌ 面板系统 / `reader.*` 全集(只闭环四动作 + state)
  - ❌ 多用户 / 多书并发 / 多会话(单会话 Mutex)
  - ❌ 一等页码 / axum·tokio
  - ❌ 印刷版 page-list 标签(留切片1+ 后续)
  - ❌ 完整 consolidation 轨迹摘要 / reader-profile 常驻上下文(留 consolidation 刀;本刀 agent 会话上下文用 `memory.recall` 兜底)
  - ❌ messages 跨会话持久化 / idle 自动判会话边界(本刀用户手动「新对话」清 messages)
  - ❌ agent 提议态做进后端 staging / memory 加 provenance 字段(可撤销落前端交互层,ADR-0030)
- **完成判据**:① 浏览器加载一本书、显示连续正文(LID 不可见);② 点四动作经 HTTP 真正驱动后端(返 effect);③ 标注从 memory 回显;④ 章节定位 + 进度% 显示;⑤ **外层 E agent 经 `/agent/chat` 可对话,reader 双向共享(agent goto 改用户视口)、动作可撤销(undo)、踪迹可见、回答分屏**;⑥(收尾)打包单进程同源启动。

## 2. A4 子切片(每步独立可验,依赖文件状态非会话内存)

> 语言归属:S10a–c·S10f = **Rust**(server/runtime);S10d·S10g = **Vue/TS**(packages/web);S10e 跨段(构建+打包)。

### S10a · server crate 骨架 + book.* 只读 GET + ts-rs API DTO 导出 `[Rust]`
- **做**:新建 `crates/server`(tiny_http、线程每连接、`Mutex<AppState{book}>`)。`book.*` 四叶子工具 → GET:`/book/manifest`、`/book/text?lid=&end=`、`/book/context?lid=&k=`、`/book/concept?name=`。错误原样透传 §4.4 分类信封(`{error_code,category,recovery?}`)。给 `QueryResponse/Citation` + 各 book 响应 + `ToolError` 信封补 `#[derive(TS)]` 导出到 `packages/web` 生成目录。
- **不做**:不接 reader/memory/query(S10b/c);不服静态资源(S10e);不接 agent(S10f)。
- **判据**:`cargo test -p server`(handler 单测用 `sample_base`)全绿 + clippy 净;手动 curl 各 GET 返正确 JSON + 错误返信封;ts-rs 生成 TS 类型文件。
- **触达**:`[ADR-0028/0021/0014/0015]`
- **实测落点**:tiny_http 路由/查询串解析样式;ts-rs 跨 crate 导出哪些类型清单。

### S10b · reader.*/memory.* POST 端点 + Reader/MemoryStore 会话状态 `[Rust]`
- **做**:`AppState` 扩 `Reader`+`MemoryStore`。POST:`/reader/goto`、`/reader/scroll`、`/reader/highlight`、`/reader/note`、`/reader/state`、`/memory/save`、`/memory/recall`(JSON body)。reader.* 返 effect;highlight/note 委托 memory.save、render 读 recall(标注单源 memory)。
- **不做**:不接 query(S10c);不做 reader 全集 / 面板。
- **判据**:`cargo test -p server` 全绿;手动 POST 驱动 reader 返 effect、标注落 memory、recall 回显;非法 LID 返 `LID_NOT_FOUND` 不降级。
- **触达**:`[ADR-0028/0015/0027/0006]`
- **实测落点**:人读视口半径(`DEFAULT_RADIUS=3` 偏小,人读窗口/无限滚动)。

### S10c · book.query POST 端点(接 runtime::query) `[Rust]`
- **做**:POST `/book/query`(body `{q, anchor_lid, scope?}`)→ 同步直调 `runtime::query`(NativeAdapter from_env)→ 返 `QueryResponse`。读模式起点:可由 `reader.state` 当前 anchor 作默认 anchor。
- **不做**:不做 synthesize;不把 query 经外层 E loop(裸调内层即完整;外层 E 在 S10f)。
- **判据**:真跑返 `QueryResponse`、citations 结构红线 100%(走 `.env`/glm-5.1,**B2 人工验**,不入自动测)。
- **触达**:`[ADR-0028/0016/0004]`
- **实测落点**:HTTP 层 query 延迟;偶发 provider 错是否在 HTTP 层重试(承 S9 内层裸调暂无重试)。

### S10d · Vue/Vite app(packages/web)+ 连续正文渲染 + 四动作 + dev proxy `[Vue/TS]`
- **做**:`packages/web`(Vue3+Vite,pnpm workspace)。import ts-rs 生成类型。组件:连续正文列(`GET /book/text` 取 `visible_lids` 渲染成无分隔连续文章)+ 四动作交互(滚动→/reader/scroll、跳转→/reader/goto、选区高亮→/reader/highlight、记笔记→/reader/note)+ memory 标注 overlay(/memory/recall)+ 读位感条(章节 + 进度%)+ 问答框(/book/query)。Vite dev proxy `/api`→tiny_http。
- **不做**:不做打包静态服(S10e);不做 agent 对话 UI(S10g);标注选区 UX 可先 CSS span,web-highlighter 类工具留实测。
- **判据**:`pnpm -C packages/web dev` 起,浏览器加载一本书显示连续正文;四动作经 HTTP 真驱动后端、标注回显、章节+进度% 显示(**B2 浏览器人工验**)。
- **触达**:`[ADR-0028/0027/0007]`
- **实测落点**:连续正文取数粒度(一次取整窗 vs 滚动渐进);标注 overlay 锚定到 DOM 的方式。

### S10f · /agent/chat 端点:外层 E agent + reader 注入 + effects/trace `[Rust]`
- **做**:改 `runtime::orchestrator::run` 签名——**删内部 `Reader::new`、改 `reader: &mut Reader` 注入**(与 `store` 对称);`OuterOutcome` 加 `effects: Vec<AgentEffect>`(本回合副作用:goto 前后 anchor、highlight/note 的 id+lid)+ `trace: Vec<TraceStep>`(tool_calls 序列摘要)。server 加 POST `/agent/chat`(body `{message}`)→ 持 `Mutex<AppState>` 把同一 `book/store/reader` 注入 `run` → 返 `{answer, incomplete, effects, trace, viewport}`。agent 标注走 `session` 层(提议态)。会话:`AppState` 持 `messages`,用户「新对话」清空(POST `/agent/new`)。
- **不做**:不做 idle 自动判会话边界;不做 consolidation 轨迹摘要(用 memory.recall 兜底);不持久化 messages 跨进程。
- **判据**:`cargo test -p server`/`-p runtime`(注入 reader 后多跳 + effects/trace 装配,FakeAdapter)全绿;真跑 `/agent/chat` 一次「问→agent goto→高亮」后 `/reader/state` 见视口已被 agent 改、effects 列出可 undo 项(**B2 人工验**)。
- **触达**:`[ADR-0030/0028/0026/0016/0015/0007]`
- **实测落点**:`AgentEffect`/`TraceStep` 字段定型;跨调用 messages 累积上限/截断;agent 标注 session 层的 layer 注入点。

### S10g · Vue agent 形态 UI:分屏 + 可撤销提议 + 踪迹 `[Vue/TS]`
- **做**:对话区(右半,分屏)接 `/agent/chat`;渲染 `effects[]` 为「提议」卡——goto「↩ 返回提问前」、highlight/note「保留(升 long_term)/撤销(memory.delete)」;渲染 `trace[]` 为可折叠查询踪迹(book.query 检索范围 + citations 链);对话末「凝练成笔记(memory.save note)/丢弃」;「新对话」按钮(/agent/new)。agent goto 驱动左侧阅读区视口同步(读 viewport)。
- **不做**:不做 reader-profile 常驻上下文展示(consolidation 刀);提议确认逐条 vs 整回合事务取简单先做、实测细化。
- **判据**:`pnpm -C packages/web dev` 浏览器:agent 回答分屏显示、agent goto 后阅读区同步且可一键返回、agent 高亮可保留/撤销、踪迹可展开(**B2 浏览器人工验**)。
- **触达**:`[ADR-0030/0028/0007]`
- **实测落点**:agent 翻页的视觉明示方式;提议卡的交互密度。

### S10e · 打包形态:tiny_http 服 dist + 一句话启动 skill `[跨段]`
- **做**:`vite build`→`dist/`;tiny_http 同端口既服 `dist/` 静态、又服 REST API + `/agent/chat`(单进程同源无 CORS)。`skills/read/`(`/understand-book:read`)拉起服务(承 ADR-0022)。
- **不做**:不做 marketplace 发布;不做多书选择 UI。
- **判据**:一句话启动 → 浏览器打开同端口 SPA + API 同源跑通;无 CORS。
- **触达**:`[ADR-0028/0022]`
- **实测落点**:tiny_http 服静态文件的 MIME/缓存样式;启动 skill 与 Rust 二进制的衔接。

## 3. 完成判据复述(本刀「完成」的充要)

```
浏览器加载书 + 连续正文(LID 隐形)
  ∧ 四动作经 HTTP 真驱动后端(返 effect)
  ∧ 标注单源 memory 回显
  ∧ 读位感(章节+进度%)
  ∧ 外层 E agent 经 /agent/chat 可对话(reader 双向共享 + 动作可撤销 + 踪迹可见 + 回答分屏)
  ∧ book.query 问答可用(结构红线 100%,B2 真跑验)
  ∧ 打包单进程同源一句话启动
```
最小连通(页面↔后端)= S10a+S10b+S10d;S10c 问答 + S10f/S10g agent + S10e 打包 = 完整画面收尾。
