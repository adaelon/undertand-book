# 参考对照与优化登记 —— 文档世界状态(Knowhere / Noire)

> 来源:冷逸《world model 与文档世界状态》(知乎,见根目录 `参考.md`)。
> 对照结论:`observation ≠ state` 的信念与本项目**同源**;文章的 Tree-like Chunking = 我们的 LID 树,Graph link = 语义边,Retrieval trace = `evidence_chain`/`citations`。我们在 LID 分区不变式 / 确定性图谱闸 / 结构红线 / Pass1·Pass2 长程边上已做得**更彻底**。
> 场景差异(企业多文档 / 版本 / 权限 / 财务口径 / 指标归一)**不纳入**——本项目是单书深读,跨书已下放 memory 层 `[ADR-0006]`、改版已有 `[ADR-0019/0020]`。
> 本文登记从该文引出的三个增量优化项。约束:只在冻结契约(V3 §6.1 基线 v1)上**只增不改**。

### §A 冲突暴露(优化项②,已采纳,切片1+)

**决策**:`book.query`/`synthesize` 遇 `contradicts` 证据时,显式暴露冲突(citation `role=contrast` + answer 点明),不抹平。

**否决**:
- 取一方 / 取多数抹平:毒化用户判断,违质量优先(memory `quality-over-speed-correct-context`)。
- 静默并列不点明:用户察觉不到冲突存在。

**命门**:仍受结构红线约束——冲突双方 `citations` 均须真 LID。
**何时回头**:依赖 `contradicts` 边质量;切片1+ 图谱补全后强化。
**展开**:`参考.md` §"评估 Agent" + `[ADR-0011]`(边作召回路标)。

### §B 证据路径可视化(优化项③,已采纳,切片1 前端)

**决策**:阅读器把 `citations`(冻结字段)渲染为可点击证据路径(点引用→跳锚定 LID 看原文);纯前端消费,不改后端契约。

**否决**:
- 只显示 answer 文本:丢可验证性,违引用红线展示面 `[ADR-0004]`。
- 后端加新接口:`citations` 已是冻结字段,前端直接消费。

**命门**:消费冻结契约已有字段;`evidence_chain` 等目标形状字段补全后再扩展。
**何时回头**:Asset 一等对象(①)落地后,证据路径可含 asset inspect。
**展开**:`[ADR-0028]`(前端切片)+ `[ADR-0030]`(agent 查询踪迹可见 = ③ 的 agent 维度)+ `docs/切片方案-切片1前端阅读器.md` S10d/S10g。

### §C Asset 一等对象(优化项①,已对齐 → `[ADR-0029]`)

**决策**:把 Asset(代码块/表/图)从被拍平的 `paragraph` 提升为**带类型的一等 LID 叶子**(`NodeKind` 增 Code/Table/Image)。

**§0.5 五问共识**(全文见 `[ADR-0029]`):
- 身份 = 带类型 LID 叶子(占 span、进分区不变式、LID 单一寻址);否决旁挂属性 / 独立 asset_id 注册表。
- 原文 = 源标记确定性序列化(image=`![alt](src)`、code 保格式、table 保表文本);否决 LLM 描述进 source。
- 读时可见 = `ManifestNode` 投影 `kind`,零新命令;`inspect_asset`=`book.text`+前端渲染。
- 类型闭集 = `{Code,Table,Image}`,长尾留实测。
- 图谱层一视同仁,确定性图谱闸不改。

**命门**:asset 占 span 是支点——分区不变式 / citation 可锚 / 图谱闸接纳全部免费获得。只增枚举/字段,不破冻结契约。
**切片归属**:独立新切片(切片1+),**不混入正在路上的「切片1 前端阅读器」那一刀**;旧基座重建后获得 asset。
