## 2. 根因:三层约束叠加

利用率(avg token / 硬闸)由三层约束**先到先触发**决定,默认 12000/80 仅 22.7% 是三者叠加的结果:

1. **软闸(叶子数)主导**(默认配置的首因)。中文书 ≈57 tok/叶 ⇒ 软闸 80 时单窗 token 天花板 ≈ 57×80 ≈ 4560,**远低于 12000 硬闸** ⇒ 软闸先触发、硬闸永远填不满。A 组放宽软闸 40→160,利用率 11.4%→38.2% 单调上升即此证。
2. **硬闸过大则稀释**。B 组软闸放开后,利用率峰值 **≈59% 落在硬闸 ≈4000**;硬闸 ≥8000 后单调下降(48%→16%),因单个章/节子树本身装不满更大的窗口。
3. **结构地板 = 34 窗口**(子树对齐 + 不跨卷的刻意代价)。B 组硬闸 16000 与 32000 **窗口数/median/max 完全相同(34 / 6468 / 12947)** ⇒ 此时每窗 = 一个完整章/节子树,不再因硬闸细分,窗口数由书的章节结构钉死。再放大硬闸只稀释利用率、不减窗口。

---

## 3. 结论

- **利用率低不是 bug,是「子树对齐 + 不跨卷」的预期成本** `[ADR-0009 决策1/命门]`。设计刻意用装箱效率换「局部边=章节内、语义边界干净」。结构地板(34 窗口)是这本书章节结构的下限,不可、也不应靠加大硬闸突破。
- **软闸≥160 后软闸退出主约束**(A 组 160→∞ 仅 38.2%→40.4%),此后由子树结构 + 硬闸主导。
- **token 装箱效率峰值 ≈59% @ 硬闸≈4000 + 软闸放开**;但这只是「填满窗口」视角的最优,**非 S2 终值**。

## 4. 对 S3 / ADR-0009 的建议(数字最终由 Pass1 抽取质量定锚)

- 当前默认 `12000/80` **偏保守**(软闸主导、调用偏多)。若 S3 实测显示 Pass1 在更大窗口下抽取质量不降,可取 **软闸 ~160 + 硬闸 ~8000**(→ ~38–45 窗口、利用率 ~40–48%),省 Pass1 调用约 30–40%。
- **最高原则**:窗口预算终值的主锚是 **Pass1 抽取质量(图谱锚定率 / 局部边质量)** 与成本的权衡,**不是 token 利用率**。利用率仅作成本侧参考——「更满 ≠ 更对」。S3 接 LLM 后用锚定率实测定 `maxInputTokens / maxLeavesSoft`,回填 [ADR-0009] 与 [ADR-0010]。
- 子树对齐与不跨卷原则在任何调参下**不变**(ADR-0009「何时回头」)。
