## 3. 锚定率 55% 的根因(关键发现)
分母 56 叶并非都"可锚":
- **非内容叶 ≈10**(本不该锚):JS 代码块(`11.18.7/9`)、维基链接(`12.10.7/8`、`13.12.10/11`)、译者注(`9.10.9`、`11.19.6/7/8`)。当前 `NodeKind` 只有 chapter/section/paragraph,**不区分代码/链接/正文** ⇒ 这些叶子进了分母却天然无实体/断言。
- 排除非内容叶后内容叶 ≈46,锚定 31 ⇒ **≈67%**。
- 剩余缺口(67% 非 100%):过渡/铺陈段(如 `9.9.4/5/7/8`)未抽断言 = **抽取召回**,非闸问题(守"不强凑断言"质量优先)。

## 4. 对全书实测 / ADR-0010·0011 的建议(数字以全书实测为准)
1. **锚定率分母应排除非内容叶**:V3 §7.2「≥90%」需明确分母 = **可锚内容叶**(排除代码/链接/注),否则代码密集书天然达不到。落点:切分层给叶子加"内容/非内容"标记(扩 NodeKind 或加 flag),或锚定率定义为「锚定叶 / 非代码非链接叶」。→ 回填 ADR-0011「何时回头」。
2. **prompt 召回**:可在 pass1 prompt 强调"每个实质论述段尽量抽一个断言",但不强凑(质量优先,宁缺悬空)。
3. **90% 目标**:按"内容叶分母"重定义后再评全书是否达标;若代码密集章系统性偏低,在 ADR-0011 标注为体裁特性而非缺陷。
