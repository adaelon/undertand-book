# LLM互动式阅读系统：需求规格说明书（V2 · 工程契约版）

> V2 由 V1.0 经对抗性压测重写而来。压测过程与逐条理由见 `docs/spec体检报告.md`(体检 15 项 + 决策 16 条)。
> V2 定位:**工程契约**。诗意/隐喻已下沉到愿景叙事层,此处只留可实现、可验收的规格。
> 凡与 V1.0 冲突处,以本文为准;括注 `[§N]` 指向体检报告中的决策编号。

## 0. 阅读指引

- 本文是**单一完整公共契约**:人类客户端与外部 Agent 消费同一套 API,后端不设特供通道 `[§7]`。
- "对外讲故事"的愿景叙事(数字感官/认知肉身等隐喻)单独放在愿景文档,不进本契约 `[§16]`。

## 1. 宪法级原则（不可违背）

- **原则一:LID 唯一寻址地基**。全书语义切分、数据引用、接口输入输出均锚定 Location ID(LID)。LID 是书的语义单元 URI 方案,保证全局唯一寻址。
  - *约束范围更正* `[§3]`:`citations[]` 中每条断言必须有真 LID;模型补充内容(无 LID)走独立字段,不受本原则约束。
- **原则二:构建期重计算,运行时锚定式推理** `[§1]`。语义切分、向量索引、知识图谱、语境胶囊在构建期固化为只读基座。运行时**允许 LLM 推理,但推理必须锚定构建期检索到的素材**;禁止脱离素材的自由臆测。
  - (V1.0 原文"运行时禁止重推理"作废——`/query`/`/synthesize`/推理代理本质即运行时推理。)
- **原则三:单一完整接口,无特供通道** `[§7]`。人与 Agent 走同一套 API,后端只产出同一份结构化 JSON;人类阅读界面是独立的前端渲染客户端,渲染不属于本契约。
  - (V1.0 原文"返回结构完全一致"中"字节同构"主张作废——保留"单一接口、无特供"内核,删除"渲染层也一致"的过度承诺。)

## 2. 系统模块与交付矩阵

| 模块 | 名称 | 核心定位 | 关键交付物 | 硬性依赖 |
| ---- | ---- | ---- | ---- | ---- |
| A | 预构建基建层 | 系统唯一只读地基 | LID 坐标表、向量索引、知识图谱、语境胶囊、OpenAPI Schema、**自检报告** | 无 |
| B | 实时消费引擎 | 计算无状态、状态外置 | 查询路由、分层消费器、外置 Session 存储 | A 的最小竖切可用 |
| C | 定位问答系统 | 业务逻辑中枢 | 意图路由、语境组装、精确引用引擎、错误契约、API 网关 | A 最小竖切 + Schema 冻结 |
| D | 开放 Agent 接口 | 标准化对外契约 | 导航/单点查询/问答三层 API、接入握手、边界 API | A 最小竖切;引用能力依赖 C |
| E | 书籍专属 Agent | 编排/状态/主动性层(**须论证增量**) | 运行时人格配置、坐标/语境/推理三代理、跨书协议 | 见 §5 欠条 |

> **模块 E 欠条** `[§7]`:E 必须论证其相对"裸调三层 API"的增量(多轮自主导航编排 / 跨 API 会话状态机 / 主动阅读策略)。若论证不成立,E 降级为"参考客户端实现",从关键路径摘除。

## 3. 模块 A：预构建只读地基

### 3.1 LID 坐标体系 `[§4]`
- **弹性有序路径**,非固定五级。LID 是深度可变的有序路径,层级随书的真实结构而定(章节书可 5 级、诗集 3 级、对话录 2 级)。
- 硬保证:① 全局唯一 ② 同级有序(可计算前后/邻接) ③ 可双向跳原文。
- 下游**不得假设固定段数**;需要"前后/跨章"时按路径前缀与序号计算。

### 3.2 导入流程 `[§5]`
- **全自动,零人工**:吃 epub/markdown/txt 的既有结构(标题层级、段落、规则切句)自动生成 LID 与切分。

### 3.3 质量自检闸（固化前强制） `[§6]`
全自动产物在写入只读基座前,必须通过确定性校验闸,**不达标自动打回或标记"低置信",不得进基座**:
- LID 唯一性断言、同级有序性断言;
- 结构自洽(无空层、无跳级);
- 切句覆盖率统计;
- 图谱锚定率统计(见 §7 验收)。

### 3.4 版本纪律 `[§9]`
- **不设版本迁移协议**。LID 导入一次即冻结;发布后基座只读。
- 书出新版/换导入算法 → **当作新书重建**(独立坐标空间),**不覆盖、不重映射**旧基座。旧 LID 永久有效。

### 3.5 其余产物
- **向量索引**:与 LID 一一绑定,构建期完成 Embedding。
- **知识图谱**:实体/关系/事件节点锚定 LID;支持 LID↔概念双向查询(确定性,毫秒级)。
- **语境胶囊**:每 LID 节点预组装"前置背景 LID 列表 + 同主题关联 LID + 概念锚点";**人设无关** `[§8]`。
- **OpenAPI 3.0 Schema**:含下游全部输入输出契约 + 错误契约(§4.5)。

## 4. 标准化 API 契约

> 两类延迟必须分清 `[§12]`:**确定性查询**(图谱/索引,毫秒级)vs **LLM 路径**(`/query`、`/synthesize`,秒级)。V1.0 把二者混称"低延迟"系认知错误。

### 4.1 导航层 `GET /manifest`
- 返回全书 LID 树状拓扑、推荐路径、段落语义权重、认知深度层级。

### 4.2 单点查询层
- `GET /context?lid={lid}&granularity={paragraph|sentence}`:返回 near(原文)/mid(前后语境 LID + 摘要)/far(关联概念锚点),片段内嵌 `citation_lid`。
- `GET /concept?name={name}`:概念定义 + 出现 LID 列表 + 关联实体。

### 4.3 问答层 `POST /query`
**请求**:
```json
{
  "session_id": "可选,状态外置",
  "query": "自然语言问题",
  "anchor_lid": "必填,当前关注位置",
  "scope": "auto|local|cross_chapter|global",
  "persona": "可选,运行时人格:academic|literary|technical",
  "include_citations": true
}
```
**响应** `[§3]`:
```json
{
  "answer": "自然语言答案",
  "citations": [
    {"lid": "12.3.1", "text": "原文片段", "role": "support|contrast"}
  ],
  "model_supplement": [
    {"text": "原文未覆盖、由模型世界知识补充的内容", "source": "model"}
  ],
  "related_concepts": ["概念A"],
  "suggested_probing": ["追问1"],
  "evidence_chain": ["推理步骤 LID 序列"],
  "scope_used": "实际检索到的范围"
}
```
**硬性规则**:
- `citations[]` **只放真 LID**;原文不足以回答时,补充内容必须进 `model_supplement`(无 LID)且显式标注 `[§2][§3]`。
- 引用准确率红线**只测 `citations[]`**(见 §7)。
- `persona` 是**运行时参数**,不改变底层数据,只改变表达风格 `[§8]`。

### 4.4 scope 路由 `[§13]`
- `auto`:**检索广度自适应**——从 `anchor_lid` 局部起,证据不足逐级外扩(节→章→跨章→全书),直到证据充分或触顶。`scope_used` 回报实际范围。
- 显式 `local|cross_chapter|global` 跳过自适应,直接锁定范围。

### 4.5 错误契约（所有端点统一） `[§14]`
```json
{
  "error_code": "LID_NOT_FOUND | LID_OUT_OF_RANGE | VERSION_MISMATCH | CONCEPT_NOT_FOUND | ...",
  "message": "人可读说明",
  "nearest_valid_lid": "可选,供 Agent 重试的最近有效坐标"
}
```
- **禁止**宽松降级(LID 不存在却静默返回最近邻内容)——会毒化引用准确率。
- 错误是契约的一部分,供 Agent 机器解析后自我纠错(支撑 §4.4 自适应检索)。

### 4.6 `POST /synthesize`
- 跨 LID 综合;请求 `anchor_lids[] + instruction`;响应同 `/query` 结构,必须显式绑定多 LID 证据;走**深路径** `[§12]`。

### 4.7 边界与接入
- `GET /limits`:速率配额、可访问深度、坐标体系版本。
- 接入握手:① 携 API Key 获取 `X-Book-Coordinate-System` → ② `GET /manifest` 对齐 LID 版本 → ③ `GET /context` 漫游 → ④ `POST /query` 消费。

## 5. 模块 E：书籍专属 Agent

### 5.1 本质与欠条
- 无自主臆测,记忆/方位/推理全部来自模块 A/D 的标准化接口。
- **立项前置**:必须写清相对裸 API 的增量(编排 / 跨 API 状态机 / 主动策略)。写不清则降级为参考客户端 `[§7]`。

### 5.2 人格 = 运行时配置 `[§8]`
- 学术 / 文学 / 技术三型为**运行时 `persona` 参数**,不在构建期烤进模块 A。一本书一套数据服务所有人格。

### 5.3 三代理机制
- 坐标代理→`/manifest`;语境代理→`/context`(消费胶囊,禁运行时重推理);推理代理→`/synthesize`(多跳,深路径)。
- **三代理若仅是三 API 的 1:1 转发,则不构成 E 的增量**——增量须落在跨代理编排与状态上(见 §5.1)。

### 5.4 跨书协议（预留）
- 书 A 的 Agent 经标准 LID 接口向书 B 查询;本书 `Manifest` 即数字名片。依赖 §3.4 的 LID 永久有效。

## 6. 实施路线（端到端竖切，取代瀑布） `[§10]`

> 废除 V1.0"顺序铁律"。改为**先打最薄竖切验证核心论点,再逐层加深**。

```
切片 0  最小样板间(端到端验证)
├── 范围:一本书 + 最小 LID(到段) + 最小图谱 + 仅 /context + /query
├── 目标:端到端跑通,验证"受限推理 over LID"在真实书上答得好不好
└── 判据:金标准集上引用准确率达标 + 人工试读认可

切片 1+  逐层加深(验证通过后)
├── 加深 LID(下探句级)、补全图谱与胶囊、加 /synthesize 深路径
├── 补 /manifest 导航层、/concept、错误契约、scope 自适应
└── 视 §5.1 欠条结论决定 E 是独立模块还是参考客户端
```

- B4 依赖倒挂(D 早于 C 却需 C 引用能力)在竖切中自动消解:首刀即含一小撮 C 的引用能力。
- Schema 冻结从"动工前四方签署"改为"切片 0 验证通过后冻结基线"。

## 7. 验收与质量红线

### 7.1 引用准确率 ≥ 95%（卡 Phase 后段） `[§11]`
- **客观锚**:小而稳的人工金标准集(问题→正确引用 LID),自动回归比对——这是**验收依据**。
- **兜底信号**:金标准未覆盖的流量用 LLM-judge,**仅作监控、不作验收**。
- 度量对象 = `citations[]` 中 LID 真实支持断言的比例。

### 7.2 图谱覆盖率 ≥ 90% `[§15]`
- **自检闸(自动)**:节点锚定率 = 有图谱锚定的 LID / 总 LID。
- **语义验收(周期)**:对金标准实体集的召回率(同 §7.1 方法论)。
- 两层都达标方可视为覆盖合格。

### 7.3 成本/延迟预算 `[§12]`
- 确定性查询(图谱/索引):毫秒级。
- LLM 快路径(简单 `/query`):目标 1–2 秒,单次小模型调用。
- LLM 深路径(多跳 / `/synthesize`):数秒,多次调用。
- (具体 p99、单查询 token/成本上限于切片 0 实测后填入。)
