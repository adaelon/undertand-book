# LLM互动式阅读系统：需求规格说明书（V3 · plugin-reader 工程契约版）

> V3 由 V2 经"以 Understand-Anything 为参照的 §0.5 领域对齐 grill"重写而来。决策依据见 `docs/adr/0001`–`0007` 与 `CONTEXT.md`(术语表)。
> V3 定位:**工程契约**。凡与 V2 冲突处,以本文为准;括注 `[ADR-N]` 指向决策记录,`[Q-N]` 指向 grill 问题。

## 0. 阅读指引

- **项目形态**:本地 Claude-plugin 类工具(类 Understand-Anything),**无托管服务端** `[ADR-0001]`。
- **两个时刻必须分清** `[ADR-0003]`:
  - **预构建期**:绑 agent harness(Claude Code / Codex 等),一次性产出**只读基座**。
  - **读时**:独立**阅读器**,与任何 harness 脱钩,LLM 后端**用户自选**。
- **"接口" = plugin 启动的 localhost 命令面**(三命名空间),不是托管 HTTP 服务 `[ADR-0001][ADR-0007]`。V2 §4 的端点语义保留,但绑 localhost。
- **参考边界**:Understand-Anything 是预构建 + 静态消费的参照;从模块 E(读时活 agent)起,U-A 不再适用,系本项目净新设计 `[ADR-0005]`。

## 1. 宪法级原则（不可违背）

- **原则一:LID 唯一寻址地基**。全书语义切分、数据引用、命令输入输出均锚定 LID。`citations[]` 中每条断言必须有真 LID;模型补充内容(无 LID)走独立字段。
- **原则二:构建期重计算,运行时锚定式推理**。语义切分、知识图谱、全局目录在构建期固化为只读基座(语境胶囊降格为读时投影,见 `[ADR-0012]`)。读时**允许推理,但必须锚定构建期检索到的素材**;读时推理由"**用户自选后端 + 我们自建的最小 agent 运行时**"执行 `[ADR-0003][ADR-0005]`。
- **原则三:人机同命令面,无特供通道** `[ADR-0007]`。人类阅读 UI、外部 Agent、内部书 agent **走同一套 localhost 命令面**;后端不设特供通道。GUI 只是命令面之上的一层渲染。

## 3. 模块 A：预构建只读基座

### 3.1 LID 坐标体系
**弹性有序路径**,非固定层级。硬保证:① 全局唯一 ② 同级有序(可计算前后/邻接)③ 可双向跳原文。下游不得假设固定段数。LID 导入一次即冻结、永久有效。
**编码与存储** `[ADR-0008]`:LID 字符串 = 物化路径(`3.2.5.2`),比较用逐段数值(非字典序);存储 = 有序 n 叉树承载层级 + 物化路径排序数组做查点/邻接/窗口范围切片(冻结只读 ⇒ 二分即可,**不引入 B/B+ 树**)。句为段下独立 LID 层,切片0 停在段、切片1+ 长出句。

### 3.2 导入流程
**全自动,零人工**:吃 epub/markdown/txt 既有结构自动生成 LID 与切分。
**切分纯确定性(LLM 零介入)** `[ADR-0008]`:段 = 忠实映射源格式块标记(md 段/li/blockquote;epub p/li/blockquote/br),不检测体裁;句 = 句末标点 `{。!?……}` + 引号/括号配对保护,对每个叶子块文本统一切出 1..N 句。裸 txt 无标记只得退化粗粒度,标低置信。

### 3.3 质量自检闸(固化前强制)
确定性校验闸,不达标自动打回或标"低置信":LID 唯一性/同级有序断言、结构自洽(无空层跳级)、切句覆盖率、图谱锚定率。
**切分校验采分区不变式** `[ADR-0008]`:全部叶子 span 对原文内容全覆盖 + 无重叠,父 span ⊇ 子并集,同级数值递增。切句覆盖率 = 入叶子内容字节/总内容字节,目标 100%(裸 txt 退化档可 <100% 但强制标低置信);非内容字节须显式归类,不准静默丢。

### 3.4 版本纪律
**不设版本迁移**。导入一次即冻结;发布后基座只读。新版/换算法 → 当新书重建,不覆盖旧基座。**只读 ⇒ 基座可分发 / 多人共享**(配合 §3.7 隔离)。
**重建可走增量优化** `[ADR-0019]`:书没变只是构建器升级、或书小改时,**以旧基座为计算缓存**只重算变更窗口(双轨变更检测:Merkle 文本树 ⊥ 层版本戳;变更分级 NONE/COSMETIC/STRUCTURAL 确定性合成;Pass2 经 catalog_hash 闸 + 受影响追踪,新机会边读时兜)。**增量 ≠ 原地更新**:产物仍是**独立新 book_id 基座、旧基座原封不动**(只读不覆盖),磁盘靠内容寻址去重。⇒ 增量是省钱的生产优化,不破本节版本纪律。记忆跨「同书新旧基座」如何延续见 §5.3(8b,留 `[ADR-0020]`)。

### 3.5 产物清单
- **知识图谱**:实体/概念/断言节点锚定 LID;语义边(builds_on/contradicts/exemplifies/cites…)。支持 LID↔概念双向查询(确定性、毫秒级)。
- **语境**(原"语境胶囊",已降格 `[ADR-0012]`):前置背景 LID + 同主题关联 LID + 概念锚点**不再预组装进基座**,改由 `book.context` 读时从最终图 + LID 树确定性投影 near/mid/far;人设无关;一切基础是图谱,不叠派生抽象层。
- **全局目录**:扁平文本索引(实体索引 + 断言索引,各带锚定 LID + 一行摘要),供长程边抽取与图谱检索入口。
- **词法索引**:Fuse.js 模糊匹配兜底(替代向量做"无概念锚点的转述"召回)`[ADR-0002]`。
- ~~向量索引~~:**已砍** `[ADR-0002]`。

### 3.6 语义边两遍抽取 `[Q-1 / A+D]` `[ADR-0010]`
书无 import/wikilink,语义边无法确定性解析,由 LLM 构建期读出,确定性闸兜底:
```
Pass 1  逐窗口抽 局部边(scope=local) + 实体/断言节点 → 沉淀「全局目录」
Pass 2  逐窗口 + 全局目录 → 抽 长程边(scope=long_range,两端跨窗口)
确定性闸  每条边两端节点存在 + 节点锚点 LID 真实存在;悬空即丢;去重
```
长程关系亦可在读时按图谱检索现场涌现(A 建索引 + D 运行时遍历)。

**两遍抽取工程化** `[ADR-0010]`(机制已定,数字留切片 0 实测):
- **双 agent**:`pass1-local-extractor` 出 `{nodes, edges(local)}`;`pass2-longrange-linker` 出 `{edges(long_range)}`、不产节点。各遍内 5 并发。
- **硬串行屏障**:全部 Pass1 完成 → 确定性 merge → 沉淀完整全局目录 → 才开 Pass2(长程边须见全量目录,不做流水线增量目录)。
- **全局目录**:由 merge 后 nodes **确定性投影**生成(非 LLM 输出),目录≡节点零幽灵条目。全量优先喂每个 Pass2 窗口;超输入硬闸则降级阶梯:**压缩**(去一行摘要)→ 压缩后仍超限则**确定性预筛**(窗内实体查目录喂 top-K)。
- **节点锚定基数**:实体/概念=单节点多锚点 `occurrences:[LID...]`(跨窗口靠 `id=entity:{name}` 归并);断言=单锚点 `source_lid`(`id=claim:{lid}:{slug}`)。
- **prompt 命门**:窗口正文每段/句**前缀 LID 标注**喂入,LLM 引用 LID 只回填不自由生成(`[ADR-0004]` 结构红线物理基础)。
- **merge + 确定性图谱闸** `[ADR-0011]`:两次 merge+闸(Pass1 后产干净中间图→投影目录;Pass2 后并入长程边→最终图)。闸做**二元判定**:锚点 LID 查 LID 全集,悬空即丢、绝不 LLM 重建;**最小连坐**(边端缺失只丢边、断言锚悬空丢节点、实体部分 occ 悬空只剔锚保节点);节点去重**按类型合并**(实体 occurrences 并集,非 keep-last)。**无 LLM 复核阶段**——纯确定性收口;构建期单条"锚点真实但关系判错"的边由读时兜(边只作召回路标,见 §4.1)。三维质量正交:锚点真实性(闸)⊥ 切分 confidence(§3.3 透传)⊥ 关系 weight。

**窗口/批次切分模型** `[ADR-0009]`(策略已定、数字留切片 0 实测):
- **窗口** = LID 子树(章/节)为基本单元;超输入预算则子树内 token 驱动细分(切点吸附 LID 边界,不腰斩句)、过小则合并相邻同级,不跨卷。⇒ 局部边=章节内、长程边=跨章节。
- **预算** = 输入硬闸(正文 token ≤ 上下文窗口×安全系数 − 指令/目录/输出预留)+ 输出软闸(单窗口节点/边软上限)。
- **执行** = 复用 harness subagent 并发:一窗口=一次抽取调用,相邻小窗口可融合、产出按区间拆回,默认 5 并发。跨窗口靠**全局目录**(书无 import,neighborMap 不适用)。
- **边界** = 子树边界零重叠(跨边界=长程边走 Pass2);仅人工细分切点加小 overlap,确定性闸去重。

### 3.7 物理隔离 `[ADR-0006]`
**只读基座 ≠ 用户记忆层**。基座冻结只读、可分发;个人记忆挂独立的、用户私有、可变层(见 §5.3),**绝不写回基座**。

## 4. 命令面契约（localhost，三命名空间）`[ADR-0001][ADR-0007]`

> 两类延迟分清 `[V2 §12]`:**确定性命令**(图谱/索引,毫秒级)vs **LLM 命令**(`book.query/synthesize`,秒级)。
> 三命名空间对应三种可变性级别:`book.*` 只读、`reader.*`/`memory.*` 可变。**硬边界:reader/memory 不得写 book 只读基座。**

### 4.1 `book.*` —— 只读内容查询(映射 ②③)`[ADR-0014]`
> **命令面分层** `[ADR-0014]`:确定性命令 = agent 的 **primitive 叶子工具**(无 LLM、可组合);LLM 命令 = **自建最小运行时(§5.2 / 议题7)被无状态调一次的暴露**。一套面即 agent tool 集,无特供 `[ADR-0007]`。`book.query/synthesize` 精确签名留议题7 与运行时同处设计,下方仅记其语义。

**确定性叶子工具(签名定型):**
- `book.manifest()` → `{ tree:[{lid, children, span}...], stats_by_lid:{lid:{child_count, leaf_count, anchored_nodes}} }`。**只暴露确定性拓扑**;砍"推荐路径/认知深度"(不可证伪,路径推荐交读时 E 现算)`[ADR-0014]`。(确定性)
- `book.context(lid, granularity ∈ {near,mid,far}, k? = <概览K> | "all" | <int>)` → `{ anchor, items:[ {lid, layer, via} ] }`。**纯指针**,不带原文/预览/节点 summary;内容走 `book.text` 取 `[ADR-0014]`。读时从最终图 + LID 树确定性投影 `[ADR-0012][ADR-0013]`。
  - **三档 = 图谱具名遍历**:near=树邻接 + `scope=local` 边;mid=经概念/实体二跳的其他 occurrences;far=`scope=long_range` 边另一端。
  - **累积 + 分层标注**:`granularity=far` 返回 near∪mid∪far,每条标 `{lid, layer, via}`;`=near` 只算近层。`via` = 判别联合(确定性接入来源 + 该档排序键):`{kind:"tree",rel}` / `{kind:"edge",scope,type,weight,direction}` / `{kind:"concept",name,shared_count}`。
  - **top-K**:默认截断(概览 K 留切片0实测;排序 near 按 LID 距离 / mid 按共享概念数 / far 按边 weight);`k="all"` 取该档全量(供 agent 凑完整上下文)`[ADR-0014]`。某概念全量亦走 `book.concept`。
- `book.text(lid, range?)` → 按 LID / LID 区间取**真原文**(纯确定性、只读、provider 无关)`[ADR-0014]`。兑现 LID 硬保证③ + 与纯指针 `book.context` 配对;落 `book.*`(原文属冻结只读基座)。(确定性)
- `book.concept(name)` → `{ concept:{name, summary}, occurrences:[lid...](全量不截断), related_entities:[...] }`。按规范化名寻址 `[ADR-0010]`;`summary` 是 Pass1 LLM 产、已冻结,**仅作展示标签,agent 推理以 occurrences 真原文为准** `[ADR-0014]`。查不到 → `CONCEPT_NOT_FOUND` + 可选 `suggestions[]`(不自动替换)。(确定性)

**LLM 命令(签名定型 `[ADR-0016][ADR-0017]`):**
- `book.query(query, anchor_lid?, scope? = "auto")`:问答(锚定推理)。`scope=auto` 的"证据不足→逐级外扩"循环**在 book.query 内部**(自含 mini-loop:确定性档位骨架捞证据 + LLM 合一轮判停作答 + 确定性交叉验停,见 §5.2 / `[ADR-0016]`),裸调即得正确完整答案 `[ADR-0014]`;显式 `scope∈{local,chapter,cross_chapter,global}` 单射定宽。**scope 四档与 `book.context` 的 granularity 三档同轴**(reduce 到同一批图谱原语,只是截断半径不同 `[ADR-0016]`)。深/快是外扩深度涌现,无独立旋钮。(LLM,签名+与 synthesize 分工定型 `[ADR-0017]`)
- `book.synthesize(lids:[LID...], task?)`:跨**给定离散 LID 集**综合(多 LID 证据绑定)`[ADR-0017]`。**按输入形态与 query 分工**:query 隐式检索+外扩、synthesize 由调用方圈定证据范围、**不外扩**。内部**确定性分批归并**(放得下单轮 / 超预算按 LID 顺序切批局部综合→LLM 归并,横向不出 lids、纵向全证据进上下文不静默丢,复用 [ADR-0009] 窗口预算)。响应**复用 query 骨架**(`answer/citations:[{lid,text,role}]/model_supplement/evidence_chain/related_concepts/suggested_probing`),`scope_used`→`source_lids`+`batched`;`citations[].role` 扩枚举(support/contrast/theme…)承载多 LID 对照,**"证据矩阵"靠 role 表达不另立结构**;`citations ⊆ 输入 lids`(结构红线范围更紧)。(LLM,深路径)

`book.query` 响应结构(引用回填机制是结构红线基础,见 §7.1):
```json
{
  "answer": "自然语言答案",
  "citations": [{"lid": "12.3.1", "text": "原文片段", "role": "support|contrast"}],
  "model_supplement": [{"text": "原文未覆盖、模型世界知识补充", "source": "model"}],
  "related_concepts": ["概念A"],
  "suggested_probing": ["追问1"],
  "evidence_chain": ["推理步骤 LID 序列"],
  "scope_used": "实际检索到的范围"
}
```
- `citations[]` **只放从检索结果回填的真 LID**,LLM 不得自由生成 LID;原文不足走 `model_supplement`(无 LID)。
- `scope=auto`:检索广度自适应——从 anchor 局部起,证据不足沿**知识图谱**逐级外扩,`scope_used` 回报实际范围 `[ADR-0002]`。
- **边作召回路标** `[ADR-0011]`:图谱遍历用**边的存在**决定捞哪些 LID 原文进证据集;`edge.type` **不当 LLM 推理强先验**,关系让 LLM 从捞回原文现判(锚定推理),type 退居召回提示 / UI 展示 / 图谱导航。⇒ 构建期错边碰不到引用真实性,最坏只污染检索精度,由"锚定真原文 + scope 外扩"两道兜底消化。

### 4.2 `reader.*` —— 可变 UI / 会话控制 `[ADR-0007][ADR-0015]`
**变更命令返回 effect(变更后相关状态),非裸 ack** —— 人机对称 + agent 闭环 + 上下文正确:
- `reader.gotoLid(lid)` / `reader.scroll(to_lid | delta)` → `{ok, viewport:{anchor_lid, visible_lids}}`
- `reader.highlight(lid, range?)` → `{ok, highlight_id}`;`reader.note(lid, text, range?)` → `{ok, note_id}`。**薄入口:持久化委托 `memory.save`,返回的 id 即记忆层 id**;reader 渲染时读 `memory.recall(anchor_lid)` 画标注(标注单一真相源 = 记忆层,防双所有者不一致)`[ADR-0006][ADR-0015]`。
- `reader.openPanel(type, ref?)` / `reader.closePanel(...)` → `{ok, panel}`
- `reader.state()` → `{viewport, open_panels, selection}`(**只读会话态**,供 agent 中途接入 / 人手动操作后 re-sync)`[ADR-0015]`。
人类能做的每个阅读器动作都在此暴露,E 与外部 Agent 经同一命令面调用。

### 4.3 `memory.*` —— 记忆层读写 `[ADR-0006][ADR-0015]`
**参考 Codex memory(`codex-rs/memories`)定型**:引用锚定(Codex `path:lines` → 我们 **LID**)+ 结构信封 + 散文 content。命令面定于此;Codex 式两阶段后台 consolidation(Phase1 抽取阅读会话 / Phase2 合并+遗忘+usage 剪枝)+ 分层渐进披露产物留议题7(运行时)`[ADR-0015]`。
```
record = { mem_id, type, layer, book_id, anchor:{lid?}|{concept?}, content,
           citations:[{lid, book_id, note}],   // 引用红线延伸:recall 可验证、可跳原文 [ADR-0004]
           usage:{count, last_used}, generated_at, source_session_id }
// type 闭集核心+可扩展 [ADR-0005 §5.4]:长期 note/highlight/qa/interest/sticking_point/journey;会话 dialogue/position
// layer ∈ {session, long_term} 显式字段
```
- `memory.save(record)` → `{ok, mem_id}`(mem_id 已存=upsert)
- `memory.recall(query{book_id?, lid?, concept?, type?, layer?, text?})` → `[record...]`(带 citations;不限 book_id 即跨书,按 concept 对齐 `[ADR-0006]`)
- `memory.delete(mem_id)` → `{ok}`(用户显式删,区别于议题7 后台 usage 遗忘)

### 4.4 错误契约(所有命令统一)`[ADR-0015]`
**分类信封 + 类型化 recovery**:
```json
{ "error_code": "LID_NOT_FOUND",
  "category": "validation | not_found | provider | budget | internal",
  "message": "人可读说明",
  "recovery": { "nearest_valid_lid": "9.1.2" } }
```
- `category` 让 agent 分流:**provider/budget = 瞬时**(可重试/换后端 `[ADR-0003]`),**not_found/validation = 永久**(改输入)。
- `recovery` 按错误类型化(`{nearest_valid_lid}` / `{suggestions:[...]}` / `{retriable, after_ms}`),**系统永不自动套用**(守禁宽松降级),仅供 Agent 自纠。
- `error_code` 闭集枚举(按类):
  - validation:`INVALID_GRANULARITY / INVALID_K / INVALID_RANGE / INVALID_SCOPE / INVALID_PANEL_TYPE / INVALID_MEMORY_TYPE`
  - not_found:`LID_NOT_FOUND / LID_OUT_OF_RANGE / CONCEPT_NOT_FOUND / MEMORY_NOT_FOUND / ANNOTATION_NOT_FOUND`
  - provider:`PROVIDER_ERROR / BACKEND_UNAVAILABLE / RATE_LIMITED`;budget:`CONTEXT_BUDGET_EXCEEDED`;internal:`INTERNAL_ERROR`
- 禁止宽松降级(LID 不存在静默返回最近邻)——会毒化引用准确率。错误是契约的一部分,供 Agent 机器解析后自我纠错。

### 4.5 provider 抽象 `[ADR-0003]`
`book.query/synthesize` 等 ③ 命令经 **provider 抽象层**对接用户自选后端(Anthropic/OpenAI/本地 Ollama…);② 命令 provider 无关。key/配置留本地 Node 进程,不进浏览器。

## 5. 模块 E：读时书 Agent `[ADR-0005][ADR-0006]`

### 5.1 本质与定位(欠条结清)
**E = 读时"这本书的 agent"**:承接用户对本书**任何**问题的**单一入口**;架在命令面之上,无自主臆测——记忆/检索/推理全部来自命令面 + 记忆层。
**真增量**(相对裸调命令面):多跳自主编排 + 会话态 + 主动阅读策略。裸调 `book.*` 是基线,E 是其上**带记忆的编排者**。

### 5.2 自建最小运行时 `[ADR-0016]`
本地查询服务内置**最小 agent loop**(工具调用 + 记忆),经 provider 抽象跑用户后端。**这是 U-A 没有、本项目净新自建的核心。** 形态 = **双层嵌套 loop**(议题7 第一叉定型):

- **外层 = E 编排 loop**(会话级,有状态 messages+memory):LLM 自主调三命名空间命令(book/reader/memory,含 `book.query/synthesize`),管多跳编排 + 会话态 + 主动策略。**双重停机**:正常停 = LLM 给最终答(无工具请求);硬闸 = `max_turns` + 累计 token 预算,触顶返 `{answer:部分答, warning:CONTEXT_BUDGET_EXCEEDED, incomplete:true}`,**不静默截断 / 不假装完整**。
- **内层 = `book.query` 自含 mini-loop**(无状态,被外层调一次,裸调即完整):**混合驱动**——确定性档位骨架管"捞什么"(scope 阶梯沿图谱确定性遍历捞 LID + 真原文),LLM 管"够不够 + 答什么";**合一轮条件产出** `{sufficient, answer?, citations?, model_supplement?}` + **确定性交叉验停**(`citations ⊆ 证据集 LID 全集` 才留 = 结构红线读时落地;零有效 citation 强制外扩堵早停;`scope==global` 触顶堵晚停)。
- **provider 适配 = 薄 ModelAdapter**:loop 控制 provider 无关恒定,只 `complete(messages, tools?, schema?) → ParsedResponse` 两实现——**NativeAdapter**(原生 tools API + JSON mode)/ **ReActAdapter**(弱后端无 tool-calling:注入 ReAct/JSON 模板 + 解析文本回填同一 ParsedResponse)。结构红线靠确定性 citations 过滤,弱后端也 100% 守 `[ADR-0004]`,只语义质量低。

`book.query/synthesize` 精确签名与分工见 §4.1(query 内层机制已定)+ 议题7 第二叉。

### 5.3 memory `[ADR-0006]`
**用户私有、跨书、两层、与只读基座物理隔离**:
- 会话工作记忆(临时):当前对话 + 阅读位置。
- 长期记忆(持久):阅读旅程、历次问答、兴趣、卡点、笔记/高亮。
- book agent 读写但不拥有;跨书实体对齐靠 LID 永久有效 + 概念层映射。
- **记录/命令模型见 §4.3**(参考 Codex memory、LID 引用锚定)`[ADR-0015]`。
- **两阶段后台 consolidation + 四层渐进披露产物 `[ADR-0018]`**(议题7 第三/四叉定型,参照 Codex `codex-rs/memories`):
  - **记忆来源三分**:显式 save(确定性用户产物 note/highlight/qa,LLM 不抽)/ Phase1 抽涌现推断(interest/sticking_point/journey,带证据→推断认知诚实标注 + 最小信号闸)/ 会话临时(dialogue/position)。
  - **Phase1(per 阅读片段)**:抽取单元 = 自上次以来未抽交互流;idle 超时(主)/ 关书·切书·退出(即时)/ 超量超时(保护)多触发取先到,后台扫 idle 够久者抽,绑 `source_session_id`。
  - **Phase2(global)**:单全局锁 + INIT/INCREMENTAL 双模式 + git baseline diff 驱动,产**四层产物**:reader-profile(常驻)/ 阅读手册(可 grep,per-book + cross-book 双维聚类)/ session 详档 / raw 原料(临时)。skills 不由 consolidation 产(= §5.4 预设工具集)。
  - **遗忘按来源分裂**:确定性用户产物只用户显式 `memory.delete` 删(永不 usage 遗忘);Phase1 推断走 usage 剪枝 + 时效遗忘 + 矛盾遗忘(推断会过时)。
  - 各阈值(idle N 分/max_unused_days/top-N)留切片0实测。
- **跨「同书新旧基座」迁移 `[ADR-0020]`**(8b,配 §3.4 增量构建):增量改版产出独立新基座 v2、旧 v1 只读并存后,v1 记忆经**消费 8a 确定性 LID diff**(`lid_migration_map`,非 [ADR-0006] 概念模糊对齐)延续到 v2。**三命运确定性重锚**:stable→投影 v2_lid;drift(原文改字)→投影 v2_lid+标「源文已变」+保留 v1 锚;removed(LID 失效)→**不猜最近邻**(守 §4.4 禁宽松降级)、标 orphaned、citation 仍指 v1(可跳 v1 原文)。**v1 记忆库永不改写**(历史事实),**迁移=recall 时读时确定性投影、不物化 v2 记忆库**(呼应 §3.5 语境不物化)。citation 带 `book_id` + v1 只读并存 ⇒ 引用红线([ADR-0004])迁移中从不破。会话临时层(position/dialogue)不迁移。

### 5.4 skills `[Q-9]`
读时**自有工具系统**(非构建期 Claude Code skill)。工具集:`book/reader/memory` 三命名空间命令 + 阅读交互(笔记/书签/"考考我"/分层讲解)。**分阶段开放**:切片先上**闭集核心**,tool 接口设计为"可扩展就绪",晚点开放第三方/用户技能。

### 5.5 人格
**运行时参数**(`academic|literary|technical`),只改表达风格,不改底层数据,不烤进基座。

### 5.6 跨书
取代 V2 §5.4 "预留":跨书 = **记忆层的自然能力** `[ADR-0006]`,book agent 经记忆层关联多本书的阅读;依赖 §3.1 LID 永久有效。

## 7. 验收与质量红线

### 7.1 引用准确率红线分层 `[ADR-0004]`
- **结构红线(后端无关,系统强制,写进契约)**:`citations[]` 只能从检索结果回填,LLM 不得自由生成 LID;确定性闸丢悬空。**任何后端(含弱模型)100% 守。**
- **语义质量(后端相关,度量+透明,不兜底)**:"LID 真支持断言"达标率用**金标准集逐后端实测**;阅读器显示"当前后端引用质量 N%",弱则提示换后端。
- (V2 "系统保证 ≥95%" 措辞作废,改为上述分层。)

### 7.2 图谱覆盖率 ≥ 90%
- 自检闸(自动):节点锚定率 = 有图谱锚定的 LID / 总 LID。
- 语义验收(周期):对金标准实体集召回率。两层都达标方合格。

### 7.3 成本/延迟预算
- 确定性命令(`book.manifest/context/concept`):毫秒级。
- LLM 快路径(简单 `book.query`):目标 1–2 秒。深路径(`book.synthesize`/多跳):数秒。
- **读时成本随用户所选后端走** `[ADR-0003]`;具体 p99、token/成本上限于切片 0 实测后填入。

### 7.4 provider 无关性验收 `[ADR-0003]`
② 命令在任意后端下行为一致(provider 无关);③ 命令在推荐后端下达 §7.1 语义质量目标。
